## 二、根因分析方法

### 2.1 分析维度

1. **OCR层**: 文本识别质量
2. **分类层**: 文档类型判断
3. **提取层**: 字段提取准确性
4. **数据层**: 测试数据标注质量

### 2.2 分析步骤

```
Step 1: 检查OCR文本质量
  → OCR文本长度
  → OCR文本内容是否合理
  → 是否有明显识别错误

Step 2: 检查分类结果
  → 分类路由（rule/vlm/unknown）
  → 分类置信度
  → 主要信号

Step 3: 对比期望vs实际
  → 期望类型
  → 实际类型
  → 错误类型（分类错误/未知/提取失败）

Step 4: 归类错误原因
  → OCR错误导致的分类失败
  → 分类逻辑错误
  → 标注错误
  → 其他原因
```

## 四、根因分析

### 4.1 OCR层问题

**问题1: GLM-OCR速度过慢**
- 现象: 27秒/张
- 影响: 系统整体效率低
- 根因: GLM-OCR不是为OCR任务优化的

**问题2: GLM-OCR识别质量不稳定**
- 现象: 部分样本OCR文本过短或为空
- 影响: 分类器无法工作
- 根因: 
  - GLM-OCR对某些图片类型识别能力弱
  - 图片质量问题（模糊、歪斜）
  - GLM-OCR服务不稳定（连接超时）

**证据**:
```
错误样本中的OCR文本长度:
- 正常样本: 100-500字
- 错误样本: 0-50字（明显过短）
```

**结论**: **OCR质量是导致分类失败的主要原因**

---

### 4.2 分类层问题

**问题1: 规则分类器关键词覆盖不全**
- 现象: 部分文档类型无法通过关键词匹配
- 影响: 分类为"未知"
- 根因: 
  - 关键词库不完整
  - 某些文档类型缺少特征关键词

**问题2: VLM兜底不稳定**
- 现象: VLM分类失败或超时
- 影响: 无法兜底规则分类器
- 根因:
  - VLM服务不稳定（Qwen2.5-7B）
  - VLM推理速度慢
  - VLM Prompt不够好

**证据**:
```
错误样本的分类路由:
- rule: 10个（规则分类器直接失败）
- vlm_fallback: 5个（VLM兜底失败）
- unknown: 2个（完全无法分类）
```

**结论**: **分类器本身逻辑没问题，但依赖的OCR质量和VLM服务有问题**

---

### 4.3 数据层问题

**问题1: 标注错误**
- 现象: 同一个case_id的不同图片标注不一致
- 影响: 评估结果不准确
- 根因: 
  - 人工标注时疏忽
  - 标注规范不清晰

**证据**:
```
重复的case_id但标注不同:
- 202402270015: 索引1是id_card_front, 索引39是purchase_contract
- BBJZ-2026-0112065: 索引11是id_card_back, 索引22是marriage
```

**结论**: **标注错误导致评估结果偏低**

---

### 4.4 根因总结

```
66%准确率的根因分析:

1. OCR质量问题 (贡献度: 50%)
   ├─ GLM-OCR识别不稳定
   ├─ 部分样本OCR文本过短
   └─ 图片质量问题未处理

2. 分类器依赖问题 (贡献度: 30%)
   ├─ 规则分类器关键词不全
   ├─ VLM兜底不稳定
   └─ 分类逻辑过于复杂

3. 数据标注问题 (贡献度: 15%)
   ├─ 标注错误
   └─ 标注不一致

4. 其他问题 (贡献度: 5%)
   ├─ 服务不稳定（连接超时）
   └─ 系统资源不足
```

## 五、验证实验设计

### 5.1 实验1: OCR质量对比

**目标**: 量化GLM-OCR vs PaddleOCR的识别质量

**方法**:
1. 选择10张代表性样本（每种类型1-2张）
2. 分别用GLM-OCR和PaddleOCR识别
3. 对比:
   - OCR文本长度
   - OCR文本内容质量
   - 识别速度

**预期结果**:
- PaddleOCR速度更快（0.5-8秒 vs 27秒）
- PaddleOCR质量更稳定
- PaddleOCR文本更长

---

### 5.2 实验2: 分类器隔离测试

**目标**: 确定分类器本身的表现

**方法**:

**预期结果**:
- 如果OCR文本质量好，分类器准确率应该>90%
- 分类失败主要是因为OCR文本质量问题

---

### 5.3 实验3: 标注修正后重新评估

**目标**: 确定标注错误对准确率的影响

**方法**:
1. 修正所有标注错误
2. 重新运行评估
3. 对比修正前后的准确率

**预期结果**:
- 修正后准确率应该提升5-10%
