**现象**: 测试结果显示"分类正确率0%"，但实际分类都正确  
**根因**: 脚本比较中文类型名（"购房合同"）和英文cert_code（"purchase_contract"）

**修复方案**:
```python
def cert_code_to_chinese(cert_code):
    """将英文cert_code转换为中文名称"""
    mapping = {
        'purchase_contract': ['购房合同', '购房合同-首页', '购房合同-内容页', '购房合同-签署页'],
        'stock_contract': ['存量房合同', '存量房合同-首页', '存量房合同-内容页', '存量房合同-签署页'],
    }
    return mapping.get(cert_code, [cert_code])
```

## 三、根因分析

### 根因1：签署页被设计为不提取字段

**代码位置**: `rule_layer.py` 第197-204行

```python
elif doc_info.doc_type in [
    DocumentType.PURCHASE_CONTRACT_STAMP,  # 购房合同签署页
    DocumentType.STOCK_CONTRACT_STAMP,      # 存量房合同签署页
    DocumentType.NOTARY_CERTIFICATE,
    DocumentType.POWER_OF_ATTORNEY,
]:
    # 签署页、公证书、委托书不需要提取字段
    fields = {k: "" for k in key_list}
```

**实际测试数据**:
- 样本7（签署页）参考字段：{合同编号, 签约日期}
- 样本9（签署页）参考字段：{} (空)

**结论**: 样本7需要提取签署页信息，但代码设计为不提取。**这是设计缺陷，不是bug**。

**影响**: 样本7准确率0%

### 根因2：内容页缺少首页字段

**现象**: 内容页只能提取到本页存在的字段（签订日期、总价款）

**样本5（内容页）**:
- 参考字段: {合同编号, 出卖人}
- 实际提取: 签订日期 2024-02-27 ✅
- **问题**: 合同编号和出卖人只在首页出现，内容页提取不到

**样本8（内容页）**:
- 参考字段: {合同编号, 总价款}
- 实际提取: 总价款 880000 ✅
- **问题**: 合同编号在首页，总价款在内容页

**结论**: 参考数据设计有问题。应该按页面类型区分参考字段，而不是所有页面都期望相同的字段。

**影响**: 样本5准确率0%，样本8准确率50%

## 七、关键发现

### 发现1：页面类型对提取的影响
- **首页**: 包含合同编号、买受人、出卖人
- **内容页**: 包含总价款、房屋地址、签订日期
- **签署页**: 包含签署日期、合同编号（但当前不提取）

**结论**: 不同页面的字段分布不同，应该按页面类型设计提取策略。

### 发现2：OCR耗时极长
- 样本4耗时1123秒（约19分钟）
- 原因：图片很大（6056行）
- 建议：考虑图片预处理（压缩、裁剪）
