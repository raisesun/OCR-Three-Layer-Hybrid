# H2 深度分析：合同签章页误判致内容页跳过提取（数据丢失）

- **分析日期**：2026-07-22
- **分析方法**：第一性原理 + 批判性思维 + 苏格拉底提问法
- **问题编号**：H2（审查报告 `docs/reviews/code_review_20260722.md`，🟠高）
- **状态**：未修复

---

## 1. 问题本质（一句话）

合同页面类型识别中，**签署页判定用了"签字/盖章/签章"等通用词且优先级最高**，导致含"签字盖章"条款的内容页/首页被误判为签署页（STAMP），进而在规则层被跳过提取、返回空字段，**造成字段数据丢失**。

---

## 2. 事实链（代码层面，已逐行验证）

### 2.1 误判源头：`_detect_contract_page_type`（classifier.py:431-465）
```python
# 签署页特征：合同签订日期、合同签订地址、签字/盖章
has_stamp_signals = any(
    kw in full_text
    for kw in ["合同签订日期", "合同签订地址", "签字", "盖章", "签章"]  # ← 通用词
)
...
# 签署页优先（但要排除监管协议）
if has_stamp_signals and not is_fund_supervision:
    return PageType.STAMP            # ← 优先级最高，任一通用词命中即判 STAMP
elif has_contract_title and has_first_page_fields:
    return PageType.FIRST_PAGE
elif has_content_fields:
    return PageType.CONTENT
else:
    return PageType.UNKNOWN
```

**问题点**：
- `"签字"/"盖章"/"签章"` 是**通用词**，在合同**任何页**都可能出现（如内容页条款"双方签字盖章生效"、首页"签字盖章"栏）
- `any(...)` 任一命中即 `has_stamp_signals=True`
- STAMP 判断**优先级最高**（先于 FIRST_PAGE/CONTENT）

### 2.2 误判后果链
```
内容页/首页含"签字盖章"条款
  → _detect_contract_page_type 返回 PageType.STAMP
  → _get_refined_doc_type: PURCHASE_CONTRACT + STAMP → PURCHASE_CONTRACT_STAMP（classifier.py:531-536）
  → rule_layer.extract: PURCHASE_CONTRACT_STAMP 在跳过列表（rule_layer.py:112-125）
  → return {k: "" for k in key_list}   # 字段全空
  → 数据丢失
```

### 2.3 rule_layer 的 STAMP 跳过（rule_layer.py:112-135）
```python
if doc_info.doc_type in [
    ...
    DocumentType.PURCHASE_CONTRACT_STAMP,
    DocumentType.STOCK_CONTRACT_STAMP,
    ...
]:
    return ExtractionResult(
        ...,
        fields={k: "" for k in key_list},   # ← 空字段
        success=True,
        error_message="封面页/盖章页，跳过提取",
    )
```
STAMP 类型被设计为"不提取"（签署页无有用字段）。逻辑本身正确，问题在**上游误判**把内容页判成 STAMP。

## 3. 根因追溯（苏格拉底自问自答）

**Q1：为什么用"签字/盖章/签章"判签署页？**
-> 签署页确实有签字盖章。但这些词太通用，不是签署页**独有**特征。

**Q2：为什么用 `any` 而非 `all`？**
-> 想放宽：签署页有其中任一特征即识别。但放宽过度，把通用词纳入，导致内容页误中。

**Q3：为什么 STAMP 优先级最高？**
-> 设计意图：确保签署页被识别（避免签署页被当内容页提取无意义字段）。但"先判 STAMP" + "通用词" 叠加，变成"含签字盖章的页都先判 STAMP"，误伤内容页/首页。

**Q4：为什么没被发现？**
-> 需要内容页恰好含"签字/盖章"字样才触发。如果测试样本的内容页不含这些词，不会暴露。这是**条件性数据丢失**，依赖具体文本，难在通用测试中覆盖。

**Q5：是设计缺陷还是演变遗留？**
-> 设计缺陷。签署页判定应基于**签署页独有特征**（如"合同签订日期/地址"），而非通用词。通用词应作为辅助信号，不能单独判定。

## 4. 批判性评估：严重性确认

agent 标 🟠高，**确认合理，且是本次审查中最严重的正确性问题之一**：
- **数据丢失**：内容页字段全空，是**确定性危害**（一旦误判）
- **触发条件**：内容页/首页含"签字/盖章/签章"。合同条款页常含"双方签字盖章"类表述，**触发概率不低**
- **隐蔽性**：`success=True` + 空字段，不易被监控发现（看似成功但无数据）

**与 H1/H5 对比**：H1 是性能（不丢数据），H5 是资源/维护（不丢数据），H2 是**数据丢失**。H2 危害性最高。

**需实际样本验证**：误判率取决于真实合同内容页含"签字/盖章"的比例。建议用基线样本量化。

## 5. 修复后的影响分析（重点）

### 5.1 正面影响
1. **消除数据丢失**：内容页/首页不再被误判 STAMP，字段正常提取
2. **合同提取准确率提升**：取决于误判率，可能显著
3. **行为可预测**：签署页判定基于独有特征，不再因通用词误触发

### 5.2 潜在风险与负面影响

1. **可能漏判真实签署页（低危害）**：收紧后，若某签署页只有"签字盖章"而无"合同签订日期/地址"，会被判 UNKNOWN/FIRST_PAGE/CONTENT 而非 STAMP。**但漏判危害远小于误判**：
   - 漏判 STAMP → 签署页被判 UNKNOWN → `_get_refined_doc_type` 返回基础类型 `PURCHASE_CONTRACT` → rule_layer 走 `extract_contract`（:180-184）提取 → **提取到无意义字段但不丢数据**
   - 误判 STAMP → 内容页跳过提取 → **字段全空，数据丢失**
   - 结论：宁可漏判 STAMP（提取无意义），不可误判内容页（丢数据）。修复方向正确

2. **UNKNOWN 增加（可接受）**：收紧后，含"签字盖章"但无强特征的页可能落 UNKNOWN。UNKNOWN → PURCHASE_CONTRACT（基础类型）→ extract_contract 提取。不丢数据，但可能提取到签署页的无意义字段（浪费，非错误）

3. **测试回归（中）**：`test_classifier.py` 可能有合同页类型测试。若测试用例依赖"签字/盖章"判 STAMP，收紧后需更新。需检查测试样本

4. **优先级调整的连锁（低）**：若同时调整优先级（先判内容页/首页再判 STAMP），需确保不破坏其他场景。建议优先用"强弱信号分离"，不轻易动优先级

### 5.3 影响范围
- **功能正确性**：消除内容页数据丢失（核心收益）
- **签署页识别**：可能略降（漏判），但不丢数据
- **测试**：需回归 `test_classifier.py` 合同分类；建议用基线样本对比修复前后
- **部署**：无影响

## 6. 方案对比

| 方案 | 改动 | 优点 | 缺点 |
|------|------|------|------|
| **A. 强弱信号分离** | 强信号（合同签订日期/地址，签署页独有）单独判 STAMP；弱信号（签字/盖章/签章）需配合强信号或排除内容页/首页特征 | 精准、保留通用词作辅助、不丢数据 | 需确认签署页强信号覆盖率 |
| **B. 多信号同时命中** | STAMP 需 ≥2 个信号命中（如"签字"+"合同签订日期"） | 简单、降低误判 | 可能漏判单信号签署页 |
| **C. 调整优先级** | 先判 FIRST_PAGE/CONTENT（强特征），再判 STAMP | 内容页优先，避免误伤 | 改变判定顺序，连锁影响大 |
| **D. 弱信号不判 STAMP** | 只有强信号（合同签订日期/地址）判 STAMP，弱信号完全移除 | 最简单、最保守 | 若签署页无强信号则全漏判 |

---

## 7. 推荐方案

**方案 A（强弱信号分离）**，理由：

1. **精准**：强信号（合同签订日期/地址）是签署页独有特征，误判率低
2. **保留辅助**：弱信号（签字/盖章）仍可用，但需强信号配合或排除内容页特征，不单独判定
3. **不丢数据**：即使漏判，UNKNOWN/CONTENT 路径都走提取（非跳过）
4. **最小改动**：仅 `_detect_contract_page_type` 内部逻辑，不动优先级/接口

**实施要点**（classifier.py:431-465）：
```python
def _detect_contract_page_type(self, full_text: str) -> PageType:
    # 强信号：签署页独有特征
    has_strong_stamp = any(
        kw in full_text for kw in ["合同签订日期", "合同签订地址"]
    )
    # 弱信号：通用词（内容页/首页也可能有）
    has_weak_stamp = any(
        kw in full_text for kw in ["签字", "盖章", "签章"]
    )
    is_fund_supervision = any(
        kw in full_text for kw in ["资金监管", "监管协议", "监管凭证", "监管资金"]
    )

    has_contract_title = any(kw in full_text for kw in ["买卖合同", "存量房买卖合同"])
    has_first_page_fields = any(kw in full_text for kw in ["卖方", "买方", "房屋坐落"])
    has_content_fields = any(kw in full_text for kw in ["房屋基本情况", "付款方式", "装修价款"])

    # STAMP 判定：强信号命中，或（弱信号命中 + 非内容页/首页特征）
    is_stamp = (has_strong_stamp or has_weak_stamp) and not is_fund_supervision
    # 排除：若有内容页/首页强特征，优先归为内容页/首页（避免误伤）
    if is_stamp and not (has_content_fields or (has_contract_title and has_first_page_fields)):
        return PageType.STAMP
    elif has_contract_title and has_first_page_fields:
        return PageType.FIRST_PAGE
    elif has_content_fields:
        return PageType.CONTENT
    else:
        return PageType.UNKNOWN
```

**关键逻辑**：STAMP 需 `is_stamp` 且**无内容页/首页强特征**。这样内容页（有 `房屋基本情况` 等）即使含"签字盖章"也不会被判 STAMP。

**需实际样本验证**（修复前）：
- 真实签署页是否都有"合同签订日期/地址"（强信号覆盖率）
- 真实内容页含"签字/盖章"的比例（误判率量化）
- 用基线样本（`scripts/regression_test_samples.py` 或 `baseline_service`）对比修复前后
