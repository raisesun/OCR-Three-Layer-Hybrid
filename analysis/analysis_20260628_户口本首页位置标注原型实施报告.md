## 二、原型迭代历程

| 版本 | 准确率 | 关键问题 | 修复方法 |
|------|--------|---------|---------|
| v1 | 0% | Y坐标范围完全错误 | 用实际OCR数据测量坐标 |
| v2 | 25% | 标签+数据合并（"别非农业家庭户"） | 添加LABEL_FRAGMENTS列表做标签剥离 |
| v3 | 50% | Y范围重叠导致行错位 | 使ROW1/ROW2范围不重叠 |
| v4 | 50% | 文档边界含右页导致坐标偏移 | 全文档算坐标+左半页过滤数据 |
| **v5** | **75%** | 左半过滤切掉长地址 | 去掉左半过滤+标签右侧同行搜索 |
| **v6** | **91.7%** | 标签+数据合并（"户号7314"） | 先检查标签本身是否包含数据 |

### 关键技术决策

1. **坐标系**：文档相对坐标（归一化到文档边界），而非图片坐标
2. **同行判断**：ROW_TOLERANCE=0.030（两行间距约0.04）
3. **地址合并**：按rcx排序拼接 + 短片段后处理修正顺序
4. **标签合并**：先检查标签item本身是否包含数据

## 三、核心技术方案（v6）

### 3.1 页面布局（文档相对坐标）

```
┌─────────────────────────────────────────┐
│ 注意事项 (0.0, 0.0, 1.0, 0.50)          │
├──────────┬──────────┬────────┬──────────┤
│ 户别(0.07)│ 非农业(0.19)│ 户主姓名(0.43)│ 王晨露(0.55)│ y∈[0.55,0.62]
├──────────┼──────────┼────────┴──────────┤
│ 户号(0.10)│ 005300251(0.19)│ 住址(0.36)│ 安徽省...(0.64)│ y∈[0.62,0.70]
├──────────┴──────────┴───────────────────┤
│ [盖章区域]  y∈[0.70,0.90]                 │
└─────────────────────────────────────────┘
```

### 3.2 提取算法流程

```
1. PP-OCRv6 全页OCR → 获取所有 (text, box, score)
2. 计算文档边界（所有text的min_x,min_y,max_x,max_y）
3. 归一化坐标：rx = (x - min_x) / doc_width
4. 检测是否首页（检查"注意事项"等关键词）
5. 对每个字段：
   a. 在Y范围内找标签（regex匹配）
   b. 如果标签item本身包含数据 → strip标签后取值
   c. 否则在标签右侧同行搜索数据items
   d. 按rcx排序合并相邻items
   e. 地址字段：后处理修正短片段顺序
```

### 3.3 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| ROW1_Y | (0.55, 0.62) | 户别+户主姓名行 |
| ROW2_Y | (0.62, 0.70) | 户号+住址行 |
| ROW_TOLERANCE | 0.030 | 同行判断容差 |
| MERGE_GAP | 0.08 | 合并间隙阈值 |
| BIG_GAP_THRESHOLD | 0.25 | 跨页边界阈值 |

## 四、剩余问题分析

### 4.1 唯一错误："王龙晨露"（OCR列合并）

**根因**：PP-OCRv6将左列"王子龙"和右列"王晨露"合并为一个item"王龙晨露"。

**证据**：
```
OCR输出: "王龙晨露" at (0.479,0.565)-(0.700,0.619) score=0.89
实际布局: 左列"王子龙" x∈[0.15,0.35]  右列"王晨露" x∈[0.55,0.75]
```

**结论**：这是OCR引擎层面的字符识别错误，位置标注无法修复。需要VLM（视觉语言模型）理解上下文才能纠正。

### 4.2 VLM兜底的价值

| 错误类型 | 位置标注能修复？ | VLM能修复？ |
|---------|----------------|------------|
| 列错位/区域混淆 | ✅ | ✅ |
| 标签+数据合并 | ✅ | ✅ |
| 长地址跨列 | ✅ | ✅ |
| **OCR字符合并** | **❌** | **✅** |
| 形近字混淆 | ❌ | ✅ |
| 数字漏识别 | ❌ | ✅ |
