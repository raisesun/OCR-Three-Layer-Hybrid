# 预处理调优与规则层优化方案

**日期**: 2026-07-02  
**问题**: 预处理（去噪+对比度增强）导致姓名和住址提取失败

---

## 1. 根因分析

### 1.1 OCR输出格式对比

| 配置 | 文本长度 | 格式特点 | 提取准确率 |
|------|----------|----------|------------|
| 无预处理 | 64字符 | 值优先，无标签结构 | 83.3% (5/6) |
| 仅去噪 | 77字符 | 标签+同行值，结构清晰 | 66.7% (4/6) ⚠ |
| 去噪+对比度 | 79字符 | 标签优先，值位置混乱 | 66.7% (4/6) ⚠ |
| 仅对比度 | 37字符 | 纯值，无标签 | 待测试 |

### 1.2 仅去噪的OCR输出
```
性别男
姓名刘开顺
公民身份号码
住址安徽省蚌埠市淮上区沫河
出生1994年3月1日
口镇汪邢村朱刘101号
340322199403014698
民族汉
```

**优点**：
- 标签和值在同一行（"姓名刘开顺"、"住址安徽省..."）
- 出生日期完整（"出生1994年3月1日"）
- 格式结构清晰

**问题**：
- 地址被分成了两行（第4行和第6行）
- 姓名提取错误（匹配到"性别男"而不是"刘开顺"）

### 1.3 姓名提取失败的根因

**正则匹配过程**：
```python
# 格式1（优先）：值+标签 "XXX\n姓名"
match = re.search(r'([一-鿿]{2,4})\s*\n\s*姓名', full_text)
# 匹配到: "性别男\n姓名" → 提取出"性别男"
```

**问题**：
1. 格式1正则匹配到"性别男\n姓名"
2. "性别男"不在排除列表中（排除的是"性别"、"民族"等）
3. 所以"性别男"被错误地当作姓名返回

**格式2正则**（本应正确匹配）：
```python
# 格式2：标签+同行值 "姓名 XXX"
match = re.search(r'(?<!户主)姓\s*名\s*[:：]?\s*([一-鿿]{2,4})(?=\s|$)', full_text)
# 能匹配到: "姓名刘开顺" → 提取出"刘开顺"
```
但因为格式1优先级更高，格式2没有被执行。

### 1.4 住址提取不完整的原因

**OCR输出**：
```
住址安徽省蚌埠市淮上区沫河
口镇汪邢村朱刘101号
```

**正则匹配**：
```python
# 格式2：标签+同行值
match = re.search(r'住址\s*([^\s]+(?:省|市|区|县|镇|乡|村|路|号|室)[^\s]*)', full_text)
# 匹配到: "住址安徽省蚌埠市淮上区沫河" → 只提取到第一行
```

**问题**：
- 地址被OCR分成两行
- 正则只匹配到标签后的第一个空格之前的内容
- 第二行"口镇汪邢村朱刘101号"没有被包含

## 2. 解决方案

### 方案A：优化规则层正则（推荐）

**修复姓名提取**：
```python
# 方法1：在排除列表中添加"性别男"等模式
if candidate not in (
    "签发机关", "性别", "民族", "出生", "住址", "公民身份号码", "有效期限",
    "性别男", "性别女", "民族汉"  # 新增
):
    fields["姓名"] = candidate

# 方法2：改进格式1正则，要求值不包含关键词
match = re.search(r'(?<!性别)(?<!民族)([一-鿿]{2,4})\s*\n\s*姓名', full_text)

# 方法3：调整格式优先级，让格式2优先（标签+同行值）
# 格式2（优先）：标签+同行值 "姓名 XXX"
match = re.search(r'(?<!户主)姓\s*名\s*[:：]?\s*([一-鿿]{2,4})(?=\s|$)', full_text)
# 格式1：值+标签 "XXX\n姓名"
if not match:
    match = re.search(r'([一-鿿]{2,4})\s*\n\s*姓名', full_text)
```

**修复住址提取**：
```python
# 支持多行地址合并
match = re.search(r'住址\s*([^\s]+(?:省|市|区|县|镇|乡|村|路|号|室)[^\n]*(?:\n[^\n]+(?:省|市|区|县|镇|乡|村|路|号|室))*)', full_text)
# 或者：匹配"住址"后面的所有地址特征词
match = re.search(r'住址\s*((?:[一-鿿]+(?:省|市|区|县|镇|乡|村|路|号|室))+)', full_text)
```

### 方案B：调优预处理参数

**仅使用去噪，禁用对比度增强**：
```python
config = OCRConfig(
    enable_image_preprocessing=True,
    preprocessing_denoise=True,
    preprocessing_deskew=False,
    preprocessing_contrast=False,  # 禁用对比度增强
    preprocessing_binarize=False,
)
```

**效果**：
- 文本长度：77字符（比无预处理多13字符）
- 格式：标签+同行值，结构清晰
- 需要配合规则层正则优化

### 方案C：混合策略

**根据图像质量自动选择预处理**：
1. 高质量图像：不预处理（当前方案）
2. 低质量图像（模糊、噪声多）：启用预处理
3. 需要实现图像质量评估逻辑
