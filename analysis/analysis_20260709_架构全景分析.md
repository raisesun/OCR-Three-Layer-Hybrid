## 三、API 请求处理全流程

```
┌──────────────────────────────────────────────────────────────────────┐
│                         HTTP CLIENT                                   │
│  POST /api/v1/ocr/async   Authorization: Bearer <api_key>            │
│  Body: multipart/form-data  files[], callback_url, priority          │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  FastAPI 中间件链                                                      │
│  ├── CORS (localhost:3000, localhost:8080)                            │
│  └── 请求日志 (method, path, status, ms)                              │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  Step 1: 认证                                                         │
│  APIKeyAuthenticator.verify(request)                                  │
│  ├── 解析 Authorization: Bearer <key>                                 │
│  ├── 校验 key ∈ OCR_API_KEYS 集合                                     │
│  └── 失败 → HTTP 401                                                  │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  Step 2: 记录 + 校验 + 保存                                           │
│  ├── record_api_call(api_key, endpoint) → INSERT api_usage            │
│  ├── 校验：文件数 ≤ 500, 扩展名 ∈ 支持列表, 单文件 ≤ 20MB             │
│  ├── 保存到 UPLOAD_DIR/<ts>_<uuid><ext>                               │
│  └── 校验失败 → HTTP 400/413, 保存失败 → HTTP 500 + 清理              │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  Step 3: 创建任务 (SQLite)                                            │
│  ├── create_task(file_count, priority, callback_url, api_key)         │
│  │   → INSERT tasks (status='pending', api_key=租户ID)                │
│  ├── add_file_result() × N                                           │
│  │   → INSERT task_results (file_size=文件大小, status='pending')      │
│  └── 返回 HTTP 202 + task_id                                         │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  Step 4: 后台 Worker (asyncio.Task)                                   │
│  TaskWorker.process(task_id, files)                                   │
│  ├── mark_processing() (WHERE status='pending' 乐观锁)                │
│  ├── 逐文件处理：                                                      │
│  │   ├── 检查取消标志 is_cancelled()                                   │
│  │   ├── asyncio.to_thread(OCRService.process_single) ← 见第四节      │
│  │   ├── update_file_result(status, result_json, timing_ms)           │
│  │   └── increment_processed()                                        │
│  └── mark_completed(total_time_ms)                                    │
└──────────────────────────────────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  Step 5: 查询结果 GET /api/v1/task/{task_id}                          │
│  ├── get_task() → SELECT * FROM tasks                                │
│  ├── get_file_results() → SELECT * FROM task_results                  │
│  ├── progress = processed_count / file_count × 100                   │
│  ├── estimated_remaining = avg_per_file × remaining_files             │
│  └── 完成后 → _build_result() 拼装 JSON                               │
└──────────────────────────────────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  Step 6: 配额查询 GET /api/v1/quota                                   │
│  ├── 本小时 API 调用次数 → COUNT api_usage                             │
│  ├── 活跃异步任务数 → COUNT tasks WHERE status IN (pending,processing) │
│  ├── 租户存储使用 → SUM(file_size) JOIN tasks ON api_key               │
│  └── 返回 {api_calls, storage, async_tasks}                           │
└──────────────────────────────────────────────────────────────────────┘
```

### 关键技术标注

| 技术点 | 实现 |
|--------|------|
| 认证 | Bearer Token, 模块级单例 APIKeyAuthenticator |
| 异步任务 | asyncio.create_task + SQLite 持久化 |
| 线程安全 | thread-local SQLite 连接 + WAL 模式 |
| 乐观锁 | WHERE status='pending' 防止 TOCTOU 竞态 |
| 租户隔离 | tasks.api_key 关联, get_quota() JOIN 查询 |
| 文件存储 | UPLOAD_DIR (环境变量 / 临时目录), 不自动清理 |

## 四、OCR 两层处理流水线

```
┌──────────────────────────────────────────────────────────────────────┐
│                         输入：文档图片                                 │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  OCR 文本提取                                                         │
│  PaddleOCR PP-OCRv6 → full_text                                      │
│  可选：图像预处理 (去噪/纠偏/对比度)                                    │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  Layer 1: 文档分类 (classifier.py)                                    │
│  KeywordDocumentClassifier — 6 级联阶段                               │
│                                                                      │
│  Stage 0: 多文档冲突检测 (买受人+出卖人+房屋类型)                       │
│  Stage 1: ★ 标准证件强信号 (常住人口登记卡→户口本, 公民身份号码→身份证)  │
│  Stage 1.5: 备选强信号 (持证人+登记日期→结婚证)                        │
│  Stage 1.6: 更多备选 (户别+户主姓名+住址关键词→户口本首页)              │
│  Stage 2: 标准单证 (发票代码+发票号码→发票)                            │
│  Stage 3: 合同字段组合 (买受人+出卖人+价款→购房合同)                    │
│  Stage 4: UNKNOWN → VLM 兜底                                         │
│                                                                      │
│  输出：DocumentInfo { doc_type, page_type, confidence, metadata }     │
│  + 页面类型检测 (封面/内容/盖章/附件)                                   │
└──────────────────────────┬───────────────────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────────────────┐
│  层路由决策                                                            │
│  已知文档类型 → RULE 层 (正则提取)                                     │
│  UNKNOWN      → VLM 层 (视觉模型)                                     │
│  OCR 文本为空 → VLM 层 (降级)                                         │
└───────┬──────────────────────────────────┬───────────────────────────┘
        │                                  │
┌───────▼───────────────────┐   ┌─────────▼───────────────────────────┐
│  Layer 2A: RULE 层         │   │  Layer 2B: VLM 层                   │
│  rule_layer.py             │   │  vlm_layer.py                       │
│                            │   │                                     │
│  ┌────────────────────┐   │   │  ┌─────────────────────────────┐   │
│  │ 文本预处理          │   │   │  │ prompt_templates.py         │   │
│  │ (去空格/标准化)     │   │   │  │ 构建结构化 Prompt            │   │
│  └─────────┬──────────┘   │   │  └──────────────┬──────────────┘   │
│            │               │   │  ┌──────────────▼──────────────┐   │
│  ┌─────────▼──────────┐   │   │  │ VLMClient.call()            │   │
│  │ 分发到领域提取器     │   │   │  │ POST localhost:8082/v1/...  │   │
│  │                    │   │   │  │ 图片 base64 + Prompt         │   │
│  │ PersonalIdExtract  │   │   │  │ temperature=0.1              │   │
│  │ HouseholdExtract   │   │   │  │ max_tokens=1024              │   │
│  │ FinancialExtract   │   │   │  │ 重试: 3x, 1s/2s/4s           │   │
│  │ AgreementExtract   │   │   │  └──────────────┬──────────────┘   │
│  │                    │   │   │  ┌──────────────▼──────────────┐   │
│  │ + PositionExtract  │   │   │  │ parse_json_from_response()  │   │
```

### 关键技术标注

| 技术点 | 实现 |
|--------|------|
| 分类策略 | 6 级联关键词路由, 置信度 0.60-0.95, 字典顺序决定优先级 |
| 正则提取 | 4 个领域提取器, 共享 regex_patterns.py, BaseExtractor 基类 |
| 位置标注 | PaddleOCR 坐标归一化, 行列定位, 3 策略提取 (标签含数据/同行搜索/直接搜索) |
| VLM 调用 | OpenAI 兼容 API, base64 图片, 指数退避重试 |
| JSON 解析 | 3 层 fallback (直接/去markdown/括号匹配) |
| 字段校验 | 正则+长度+字符类型+地址关键词, INVALID 触发VLM重试 |
| 多页合并 | merge_fields_first_nonempty(), 首个非空值胜出 |

## 五、数据持久化架构

### SQLite 表结构

```sql
-- 任务表
tasks (
    id              TEXT PRIMARY KEY,       -- "task_YYYYMMDD_<uuid12>"
    status          TEXT,                   -- pending|processing|completed|cancelled|failed
    file_count      INTEGER,
    processed_count INTEGER,
    priority        TEXT,                   -- normal|urgent
    callback_url    TEXT,
    created_at      TEXT,                   -- ISO-8601
    started_at      TEXT,
    completed_at    TEXT,
    total_time_ms   INTEGER,
    error_message   TEXT,                   -- 迁移新增
    api_key         TEXT                    -- 迁移新增 (租户ID)
)

-- 文件结果表
task_results (
    id              INTEGER PRIMARY KEY,
    task_id         TEXT,                   -- FK → tasks.id
    file_name       TEXT,
    file_path       TEXT,                   -- 绝对路径
    file_size       INTEGER DEFAULT 0,      -- 迁移新增 (字节)
    status          TEXT,                   -- pending|success|failed
    result_json     TEXT,                   -- JSON 提取结果
    error_message   TEXT,
    processed_at    TEXT,
    timing_ms       INTEGER
)

-- API 调用记录表
api_usage (
    id        INTEGER PRIMARY KEY,
    api_key   TEXT,
    endpoint  TEXT,
    called_at TEXT                          -- ISO-8601
)
```

### 租户配额计算（纯数据库查询，无文件系统扫描）

```sql
-- 存储使用量 (JOIN 查询)
SELECT COALESCE(SUM(tr.file_size), 0)
FROM task_results tr JOIN tasks t ON tr.task_id = t.id
WHERE t.api_key = ?

-- 活跃任务数
SELECT COUNT(*) FROM tasks
WHERE api_key = ? AND status IN ('pending', 'processing')

-- 本小时 API 调用
SELECT COUNT(*) FROM api_usage
WHERE api_key = ? AND called_at >= ?
```
