### 关键技术点

1. **三种 OCR 引擎**：
   - **PP-OCRv6**：快速稳定，41.5秒/张（包含预处理）
   - **GLM-OCR**：视觉模型，27秒/张
   - **PaddleOCR-VL**：精度高，151秒/张

2. **分层策略（tiered）**：
   - 阶段1：PP-OCRv6（快速）
   - 阶段2：如果文本 <100字，使用 PaddleOCR-VL 重新识别
   - **测试结果**：准确率 68%（失败），速度 66.9秒（慢 61%）

3. **引擎映射**：
   ```python
   engine_map = {
       "ppocr": "ppocr",           # PP-OCRv6
       "paddleocr_vl": "vlm",      # PaddleOCR-VL
       "structure_v3": "structure_v3",  # PP-StructureV3（弃用）
   }
   ```

### 关键技术点

1. **混合分类器（HybridDocumentClassifier）**：
   - **规则层优先**：KeywordDocumentClassifier（快速、无幻觉）
   - **VLM 兜底**：VLMDocumentClassifier（理解能力强）

2. **Pipeline 多阶段路由**：
   ```
   阶段0: 多文档冲突检测（买受人+出卖人+房屋类型）
   ↓ 失败
   阶段1: 标准证件强信号（公民身份号码、常住人口登记卡）
   ↓ 失败
   阶段1.5: 备选强信号（户口簿+户主、持证人+登记日期）
   ↓ 失败
   阶段1.6: 更多备选信号（户别+户主姓名、结婚证+登记机关）
   ↓ 失败
   阶段2: 标准单证强信号（发票代码+发票号码）
   ↓ 失败
   阶段3: 合同字段组合（买受人+出卖人+价款）
   ↓ 失败
   阶段4: VLM 分类兜底（Qwen3.5-4B 视觉识别）
   ```

3. **分类结果**：
   ```python
   DocumentInfo:
   - doc_type: 文档类型（身份证、户口本、购房合同等）
   - confidence: 置信度（0-1）
   - metadata:
     - route: 匹配的路由（stage0, stage1, stage2, etc.）
     - signal: 匹配的信号
     - primary_signals: 主要信号列表
     - vlm_result: VLM 分类结果（如果触发）
   ```

4. **VLM 分类服务**：
   - **模型**：Qwen3.5-4B-Uncensored
   - **端口**：8081
   - **用途**：规则层失败时的兜底分类

## 四、步骤3: 字段提取（详细流程）

```
┌─────────────────────────────────────────────────────────────────┐
│                    字段提取流程 (Pipeline)                        │
└─────────────────────────────────────────────────────────────────┘

                    ┌─────────────────┐
                    │  输入:           │
                    │  - image_path    │
                    │  - ocr_texts     │
                    │  - doc_info      │
                    └─────────────────┘
                              │
                              ▼
            ┌─────────────────────────────────┐
            │  PlanEPlusPipeline              │
            │  (多阶段路由 + 三层提取)         │
            └─────────────────────────────────┘
                              │
                              ▼
            ┌─────────────────────────────────┐
            │  根据 doc_info.route 确定提取层  │
            └─────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              │               │               │
              ▼               ▼               ▼
    ┌─────────────┐  ┌─────────────┐  ┌─────────────┐
    │ 规则层      │  │ VLM 层      │  │ 位置标注    │
    │ (Rule Layer)│  │ (VLM Layer) │  │ 提取器      │
    └─────────────┘  └─────────────┘  └─────────────┘
              │               │               │
              ▼               ▼               ▼
    ┌─────────────────────────────────────────────────┐
    │  规则层提取 (RuleExtractionLayer)                │
    │                                                  │
    │  输入:                                           │
    │  - doc_info (文档类型、路由)                     │
    │  - ocr_texts (OCR 文本)                          │
    │  - image_path (图片路径，用于位置标注)           │
    │                                                  │
    │  流程:                                           │
    │  1. 根据 doc_type 选择提取规则                   │
    │  2. 使用正则表达式提取字段                       │
    │  3. 如果是户口本首页，使用位置标注提取器         │
    │  4. 返回提取的字段                               │
    │                                                  │
    │  示例:                                           │
    │  - 身份证: 正则提取姓名、身份证号、地址等       │
    │  - 户口本: 位置标注提取户别、户主、住址等       │
    │  - 发票: 正则提取发票代码、金额、日期等         │
    └─────────────────────────────────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │ 提取成功?       │
                    │ (字段数量 > 0)  │
                    └─────────────────┘
                              │
                    ┌─────────┴─────────┐
                    │                   │
                   是                  否
                    │                   │
                    ▼                   ▼
          ┌─────────────┐     ┌─────────────────┐
          │ 返回提取结果 │     │ VLM 层提取      │
          │ fields      │     │ (VLMExtraction  │
          │ layer=rule  │     │  Layer)         │
          └─────────────┘     └─────────────────┘
                                        │
                                        ▼
                              ┌─────────────────┐
                              │ VLM 层提取       │
                              │                  │
                              │ 使用 GLM-OCR     │
                              │ (端口 8080)      │
                              │ 视觉模型提取字段 │
                              │                  │
                              │ Prompt:          │
                              │ "请从图片中提取  │
                              │  以下字段: ..."  │
                              └─────────────────┘
                                        │
                                        ▼
                              ┌─────────────────┐
                              │ 返回提取结果     │
                              │ fields          │
                              │ layer=vlm       │
                              └─────────────────┘
```

### 关键技术点

1. **三层提取架构**：
   ```
   第1层: 规则层 (Rule Layer)
   - 使用正则表达式提取
   - 速度快（<0.1秒）
   - 无幻觉
   - 适用：固定格式文档（身份证、户口本、发票等）
   
   第2层: VLM 层 (VLM Layer)
   - 使用 GLM-OCR 视觉模型
   - 速度中等（10-30秒）
   - 理解能力强
   - 适用：规则层失败时（OCR 错误、格式变化）
   
   第3层: 位置标注提取器 (Position Extractor)
   - 使用 PaddleOCR 坐标信息
   - 针对户口本首页优化
   - 通过坐标位置识别字段（户别、户主、住址等）
   - 适用：户口本首页
   ```

2. **位置标注提取器（HouseholdPositionExtractor）**：
   ```python
   流程:
   1. 使用 PaddleOCR 检测文本框坐标
   2. 根据坐标位置判断字段类型
      - 左上角: 户别
      - 右上角: 户号
      - 中间: 户主姓名
      - 下方: 住址
   3. 提取对应区域的文本
   4. 返回字段字典
   
   优势:
   - 不受 OCR 文本顺序影响
   - 通过坐标位置精确定位字段
   - 解决列合并问题（如"王龙晨露"）
   ```

3. **VLM 提取服务**：
   - **模型**：GLM-OCR-Q8_0.gguf
   - **端口**：8080
   - **用途**：规则层失败时的兜底提取
   - **Prompt 示例**：
     ```
     请从图片中提取以下字段:
     - 姓名
     - 身份证号
     - 地址
     直接输出 JSON 格式。
     ```

4. **提取结果**：
   ```python
   ExtractionResult:
   - fields: 提取的字段字典
     {
       "姓名": "张三",
       "身份证号": "340321199210127956",
       "地址": "安徽省蚌埠市..."
     }
   - layer: 使用的提取层（rule, vlm, none）
   - success: 是否成功
   - error_message: 错误信息（如果失败）
   ```

## 七、关键技术点总结

### 1. 分类技术

| 技术 | 说明 | 优势 | 劣势 |
|------|------|------|------|
| **规则层分类** | 关键词匹配 + Pipeline 路由 | 快速、无幻觉 | 无法处理 OCR 错误 |
| **VLM 分类** | Qwen3.5-4B 视觉识别 | 理解能力强 | 速度较慢（5-15秒） |
| **混合分类** | 规则层优先 + VLM 兜底 | 兼顾速度和准确率 | 实现复杂 |

### 2. 提取技术

| 技术 | 说明 | 优势 | 劣势 |
|------|------|------|------|
| **规则层提取** | 正则表达式 | 速度快（<0.1秒） | 无法处理 OCR 错误 |
| **VLM 提取** | GLM-OCR 视觉模型 | 理解能力强 | 速度较慢（10-30秒） |
| **位置标注提取** | PaddleOCR 坐标 | 精确定位字段 | 仅适用于特定文档 |

### 3. 分层策略

| 层级 | 引擎 | 速度 | 准确率 | 适用场景 |
|------|------|------|--------|---------|
| **OCR 层** | PP-OCRv6 | 41.5秒 | 70% | A/B 级文档 |
| **OCR 层** | GLM-OCR | 27秒 | 66% | 快速场景 |
| **OCR 层** | PaddleOCR-VL | 151秒 | - | C/D 级文档 |
| **提取层** | 规则层 | <0.1秒 | 高 | 固定格式 |
| **提取层** | VLM 层 | 10-30秒 | 中 | 复杂格式 |

### 4. Pipeline 路由

```
阶段0: 多文档冲突检测
↓ 失败
阶段1: 标准证件强信号（公民身份号码、常住人口登记卡）
↓ 失败
阶段1.5: 备选强信号（户口簿+户主、持证人+登记日期）
↓ 失败
阶段1.6: 更多备选信号（户别+户主姓名、结婚证+登记机关）
↓ 失败
阶段2: 标准单证强信号（发票代码+发票号码）
↓ 失败
阶段3: 合同字段组合（买受人+出卖人+价款）
↓ 失败
阶段4: VLM 分类兜底（Qwen3.5-4B 视觉识别）
```
