## 详细执行步骤

### Step 1: 服务初始化

**文件**: `service.py`
**方法**: `OCRService.__init__()`

```
输入: config (可选)
  │
  ├─ 创建 VLMClient (连接GLM-OCR或Qwen2.5-VL-7B)
  ├─ 创建 KeywordDocumentClassifier (规则分类器)
  ├─ 创建 HouseholdPositionExtractor (可选，位置标注提取)
  ├─ 创建 VLMFallbackHandler (可选，VLM字段级兜底)
  ├─ 创建 RuleExtractionLayer (注入位置标注提取器)
  ├─ 创建 VLMExtractionLayer
  └─ 创建 PlanEPlusPipeline (注入分类器、规则层、VLM层、兜底处理器)
```

**关键技术**:
- 依赖注入：各组件通过构造函数注入，便于测试和替换
- 延迟初始化：PaddleOCR在首次调用时初始化
- 配置驱动：通过 `OCRConfig` 控制启用哪些功能

---

### Step 2: OCR文本提取

**文件**: `service.py`
**方法**: `OCRService.run_ocr()`

```
输入: image_path (图片路径)
  │
  ├─ [可选] 图像预处理 (image_preprocessor.py::enhance_image)
  │    - 去噪、纠偏、增强对比度、二值化
  │
  ├─ 调用 PaddleOCRWrapper.run_ocr()
  │    └─ 使用 PP-OCRv6 引擎
  │         - 文本检测 (DB算法)
  │         - 文本识别 (CRNN算法)
  │         - 位置坐标返回
  │
  └─ 输出: full_text (合并后的OCR文本)
```

**核心技术**:
| 组件 | 技术 | 说明 |
|------|------|------|
| 文本检测 | DB (Differentiable Binarization) | 自适应阈值二值化 |
| 文本识别 | CRNN (CNN+RNN+CTC) | 支持中英文混合 |
| 位置感知 | PaddleOCR坐标返回 | 用于后续位置标注提取 |
| 图像预处理 | OpenCV | 可选，提升低质量图片OCR效果 |

**性能**: 单张图片约 2-5 秒

---

### Step 3: 文档分类

**文件**: `classifier.py`
**方法**: `KeywordDocumentClassifier.classify()`

```
输入: image_path, ocr_texts
  │
  ├─ 文本预处理: "".join(ocr_texts).replace(" ", "")
  │    ─ 修复OCR空格问题 ("结 婚" → "结婚")
  │
  ├─ Stage 0: 多文档冲突检测
  │    └─ 检测: 买受人 + 出卖人 + (商品房|存量房)
  │         └─ 命中 → 直接返回合同类型 (优先级最高)
  │
  ├─ Stage 1: 标准证件强信号
  │    └─ 遍历 STANDARD_CERTIFICATE_SIGNALS
  │         - 户口本: "常住人口登记卡"
  │         - 身份证: "公民身份号码"
  │         - 结婚证: "结婚证字号"
  │         - 离婚证: "离婚证字号"
  │         - 房产证: "不动产权证书"
  │         - 发票: "发票代码"+"发票号码"
  │         ─ 命中 → 返回对应类型 (confidence=0.95)
  │
  ├─ Stage 1.5: 备选强信号
  │    └─ 遍历 BACKUP_CERTIFICATE_SIGNALS
  │         - 结婚证: "持证人" + "登记日期"
  │         - 户口本: "户口簿" + "户主"
  │         - 身份证: "公民身份" + 18位号码正则
  │         └─ 命中 → 返回对应类型 (confidence=0.90)
  │
  ├─ Stage 1.6: 更多备选信号
  │    └─ 遍历 ADDITIONAL_BACKUP_SIGNALS
  │         - 户口本首页: "户别" + "户主姓名" + "住址"
  │         - 结婚证盖章页: "结婚证" + "登记机关"
  │         ─ 命中 → 返回对应类型 (confidence=0.85)
  │
  ├─ Stage 2: 标准单证强信号
  │    └─ 发票: "发票代码" + "发票号码" 同时存在
  │         └─ 命中 → 返回 INVOICE (confidence=0.95)
  │
  ├─ Stage 3: 合同字段组合
  │    └─ 遍历 CONTRACT_SIGNALS
  │         - 购房合同: 买受人 + 出卖人 + 商品房
  │         - 存量房合同: 买受人 + 出卖人 + 存量房
  │         - 资金监管协议: 资金监管 + 监管协议/金额
  │         - 离婚协议: 离婚 + 财产分割/抚养
  │         └─ 命中 → 返回对应类型 (confidence=0.85-0.90)
  │
  └─ Stage 4: VLM兜底
       └─ 未命中任何规则 → 返回 UNKNOWN (confidence=0.0)
```

**页面类型细化** (在基础分类后执行):

```
方法: _detect_page_type()
  │
  ├─ 离婚证/结婚证: 封面页/内容页/盖章页
  ├─ 户口本: 首页/个人页
  ├─ 房产证: 首页/内容页/附图页
  ├─ 身份证: 正面/背面
  ├─ 资金监管协议: 首页/信息页/签章页
  └─ 购房/存量房合同: 首页/内容页/签署页

方法: _get_refined_doc_type()
  └─ 将基础类型 + 页面类型 → 细化类型
       例: MARRIAGE_CERTIFICATE + CONTENT → MARRIAGE_CERTIFICATE_CONTENT
```

**关键技术**:
| 技术 | 说明 |
|------|------|
| 多阶段路由 | 4个阶段，优先级递减，快速匹配 |
| 关键词匹配 | 基于领域知识的精确关键词 |
| 组合信号 | 多个关键词组合提高准确率 |
| 正则表达式 | 18位身份证号验证 |
| OCR空格修复 | 去除空格处理破碎OCR文本 |

**性能**: 单张图片约 1-5 毫秒

### Step 4: 字段提取 (Pipeline编排 + 提取层)

**文件**: `pipeline.py`
**方法**: `PlanEPlusPipeline.process()`

```
输入: image_path, ocr_texts, doc_info (预分类结果)
  │
  ├─ 获取字段列表: key_list = DEFAULT_KEY_LISTS[doc_type]
  │
  ├─ 选择处理层: layer = DEFAULT_LAYER_ROUTING[doc_type]
  │    ─ 当前配置: 所有已知类型 → RULE, UNKNOWN → VLM
  │
  ├─ 回退逻辑: 若RULE层但无OCR文本 → 回退到VLM
  │
  ├─ 调用 layer.extract(doc_info, key_list)
  │    │
  │    ├─ [分支1] RULE层 → rule_layer.py::RuleExtractionLayer.extract()
  │    │    │
  │    │    ├─ 文本预处理: preprocess_text()
  │    │    ├─ 封面页/盖章页检测 → 返回空字段
  │    │    │
  │    │    ├─ 根据doc_type路由到对应提取方法:
  │    │    │    ├─ _extract_id_card()          # 身份证
  │    │    │    ├─ _extract_household_register() # 户口本 (位置标注+正则)
  │    │    │    ├─ _extract_marriage_certificate() # 结婚证
  │    │    │    ├─ _extract_divorce_certificate()  # 离婚证
  │    │    │    ├─ _extract_property_certificate_content() # 房产证
  │    │    │    ├─ _extract_invoice()            # 发票
  │    │    │    ├─ _extract_contract()           # 购房/存量房合同
  │    │    │    ├─ _extract_fund_supervision()   # 资金监管协议
  │    │    │    └─ _extract_fund_supervision_certificate() # 资金监管凭证
  │    │    │
  │    │    ├─ 关键技术: 正则表达式、位置标注提取、文本预处理
  │    │    └─ 性能: 10-50 毫秒
  │    │
  │    └─ [分支2] VLM层 → vlm_layer.py::VLMExtractionLayer.extract()
  │         │
  │         ├─ 选择Prompt模板: PROMPT_TEMPLATES[doc_type]
  │         ├─ 调用 VLMClient.chat()
  │         │    ├─ 图片: 原始图片文件
  │         │    ├─ Prompt: 专用模板 + 字段列表
  │         │    └─ 模型: GLM-OCR 或 Qwen2.5-VL-7B
  │         ├─ 解析VLM响应 (JSON格式)
  │         │
  │         ├─ 关键技术: 多模态大模型、专用Prompt、JSON输出
  │         └─ 性能: 30-120 秒
  │
  ├─ VLM字段级兜底 (可选，仅对户口本/结婚证/身份证)
  │    └─ 校验提取结果 → 失败字段触发VLM重新提取 → 合并结果
  │
  └─ 输出: ExtractionResult
```

**路由表** (DEFAULT_LAYER_ROUTING):

| 文档类型 | 处理层 | 说明 |
|----------|--------|------|
| 身份证/正面/背面 | RULE | 正则提取 |
| 结婚证 (所有页面) | RULE | 正则提取 |
| 离婚证 (所有页面) | RULE | 正则提取 + VLM兜底 |
| 户口本 (所有页面) | RULE | 位置标注+正则 + VLM兜底 |
| 房产证 (所有页面) | RULE | 正则提取 |
| 发票 | RULE | 正则提取 |
| 购房合同 (所有页面) | RULE | 正则提取 |
| 存量房合同 (所有页面) | RULE | 正则提取 |
| 资金监管协议/凭证 | RULE | 正则提取 |
| 离婚协议 | RULE | 正则提取 |
| 公证书/委托书 | RULE | 返回空字段 |
| UNKNOWN | VLM | VLM多模态提取 |

### Step 5: VLM字段级兜底 (可选)

**文件**: `pipeline.py`
**方法**: `PlanEPlusPipeline._apply_vlm_fallback()`

```
触发条件:
  ├─ 文档类型 ∈ {户口本, 结婚证, 身份证}
  ├─ 规则层提取成功
  └─ 字段校验失败 (如: 身份证号不是18位)

执行流程:
  │
  ├─ 调用 vlm_fallback_handler.get_failed_fields()
  │    └─ 校验每个字段的格式
  │         - 身份证号: 18位正则
  │         - 日期: 格式验证
  │         - 其他: 非空检查
  │
  ├─ 若有失败字段 → 调用 vlm_fallback_handler.fallback_extract()
  │    ├─ 构建VLM Prompt (仅针对失败字段)
  │    ├─ 调用VLM重新提取
  │    └─ 返回失败字段的VLM提取结果
  │
  ├─ 合并结果: 用VLM结果覆盖失败字段
  │
  ─ 标记: result.vlm_fallback_triggered = True
```

**关键技术**:
| 技术 | 说明 |
|------|------|
| 字段校验器 | 23个字段的校验规则 |
| 选择性兜底 | 仅对失败字段触发VLM |
| 结果合并 | VLM结果只覆盖失败字段 |

## 文件依赖关系图

```
service.py (统一入口)
    │
    ├── classifier.py (文档分类)
    │     └── interfaces.py (DocumentType, DocumentInfo, PageType)

    │     ├── interfaces.py (ExtractionResult, ProcessingLayer)

    │
    └── paddleocr_wrapper.py (OCR引擎)

```

## 核心配置

**文件**: `config.py`

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `vlm_extraction_engine` | "glm_ocr" | VLM引擎选择 |
| `enable_position_extraction` | True | 启用位置标注提取 |
| `enable_vlm_field_fallback` | True | 启用VLM字段级兜底 |
| `enable_image_preprocessing` | False | 启用图像预处理 |
| `vlm_service.base_url` | "http://localhost:8080" | GLM-OCR服务地址 |
| `qwen_vl_service.base_url` | "http://localhost:8082" | Qwen2.5-VL服务地址 |

## 性能总结

| 阶段 | 平均耗时 | 关键技术 |
|------|----------|----------|
| OCR文本提取 | 2-5秒 | PP-OCRv6 |
| 文档分类 | 1-5毫秒 | 关键词匹配 |
| 规则层提取 | 10-50毫秒 | 正则表达式 |
| VLM层提取 | 30-120秒 | GLM-OCR/Qwen2.5-VL |
| **总体 (规则层)** | **3-6秒** | OCR+分类+规则 |
| **总体 (VLM层)** | **30-120秒** | OCR+分类+VLM |

## 扩展点

1. **新增文档类型**: 
   - 在 `interfaces.py::DocumentType` 添加枚举
   - 在 `classifier.py` 添加分类规则
   - 在 `rule_layer.py` 或 `vlm_layer.py` 添加提取逻辑

2. **替换OCR引擎**: 
   - 修改 `paddleocr_wrapper.py` 或实现新的wrapper

3. **替换VLM模型**: 
   - 修改 `config.py` 中的服务地址
   - 在 `vlm_layer.py` 中添加新模型的Prompt模板

4. **添加新的字段校验规则**: 
   - 修改 `vlm_fallback.py` 中的校验逻辑
