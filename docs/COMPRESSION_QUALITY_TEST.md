# 压缩质量测试说明

**创建日期**: 2026-07-02  
**状态**: 进行中

---

## 一、测试目标

找到**文件大小**与**OCR准确率**的最佳平衡点。

## 二、测试参数

**测试质量**: 95%, 85%, 75%, 65%, 55%

**测试样本**: 5个（从batch_test_50_samples.json中选取）

**测试指标**:
- 平均字段数（衡量准确率）
- 平均耗时（衡量性能）
- 平均文件大小（衡量存储成本）

## 三、预期结果

### 假设
- 95%：质量最高，文件最大，准确率最高
- 75-85%：质量损失很小，文件大小显著减少
- 55-65%：质量明显下降，准确率可能下降

### 期望结论
- 最佳平衡点可能在**75-85%**
- 文件大小可减少**30-50%**
- 准确率损失在可接受范围内（< 10%）

## 四、测试脚本

**文件**: `scripts/test_compression_quality.py`

**使用方法**:
```bash
python3 scripts/test_compression_quality.py
```

**输出**:
- 每种质量的详细测试结果
- 对比表格
- 最佳平衡点推荐

## 五、实施建议

### 如果75-85%效果好
- 更新默认JPEG质量为85%
- 在`image_preprocessor.py`中修改`quality`参数
- 预期收益：文件大小减少30-40%，准确率基本不变

### 如果65%以下效果差
- 不建议使用低于75%的质量
- 对于需要高准确率的场景，保持95%质量

## 六、后续优化

1. **自适应压缩**
   - 根据图片内容自动选择压缩质量
   - 文字密集的图片：高压缩质量（85-95%）
   - 简单文档：低压缩质量（65-75%）

2. **智能压缩**
   - 使用WebP格式替代JPEG
   - 相同质量下，文件更小
   - 需要评估兼容性

3. **分区域压缩**
   - 文字区域：高质量
   - 背景区域：低质量
   - 需要图像分割技术支持
