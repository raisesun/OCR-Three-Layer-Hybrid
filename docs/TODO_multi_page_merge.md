# 待办：多页文档合并架构

## 背景
户口本、购房合同等文档可能跨越多张图片（多页），当前系统逐页独立处理，
需要将多页数据合并到统一的文档级数据结构中。

## 需求
1. 户口本：首页（户主信息）+ 内容页（家庭成员个人页面）→ 结构化层级记录
2. 购房合同：封面页 + 当事人页 + 房屋信息页 + 签章页 → 合并为一份合同记录
3. 资金监管协议：多页 → 合并为一份协议记录

## 设计要点
- 页面分类（首页/内容页/签章页）的识别方法
- 合并策略（首页字段 + 内容页字段 → 文档级记录）
- 输出格式（结构化嵌套 vs 扁平字段表）
- 与当前 per-image 处理的兼容性
