## 3. 实施内容

### 3.1 修复清单

#### ✅ 修复1: 签署日期
**问题**: OCR缺少"日"字 + 格式差异（无标签）  
**解决方案**: 添加 `于XXXX年X月X签订` 模式，自动补充"日"字

```python
# 模式2: "于XXXX年X月X签订" (处理OCR缺少"日"字的情况)
match = re.search(r'于\s*(\d{4}年\d{1,2}月\d{1,2})\s*签订', full_text)
if match:
    value = match.group(1) + "日"  # 补充"日"字
```

**效果**: ✅ 成功提取 "2026年1月12日"

---

#### ✅ 修复2: 网上签约备案合同号
**问题**: 文档使用 "为：" 而正则只匹配 "："  
**解决方案**: 添加 `(?:为)?` 使"为"可选

```python
match = re.search(r'(?:网上签约备案合同号|备案合同号|合同号)\s*(?:为)?\s*[:：]?\s*([A-Z0-9\-()（）]+)', full_text, re.IGNORECASE)
```

**效果**: ✅ 成功提取 "BBJZ-2026-0112065"

---

#### ✅ 修复3: 房屋地址
**问题**: 文档使用 "位于xxx" 而不是 "房屋地址：xxx"  
**解决方案**: 添加 `位于xxx` 模式

```python
# 模式2: "位于xxx"
match = re.search(r'位于\s*([^\n，,。]+(?:号|室|栋|楼|单元|层))', full_text)
```

**效果**: ✅ 成功提取 "蚌埠市安瑞科嘉苑1号楼1单元11层4号"

---

#### ✅ 修复4: 不动产权证号
**问题**: 文本跨行导致正则无法匹配  
**解决方案**: 添加 `re.DOTALL` 标志支持跨行匹配

```python
match = re.search(
    r'[一-龥]*\s*[（(]\s*\d{4}\s*[）)]\s*[一-龥]+\s*市?\s*不动产权第\s*[A-Z0-9]+\s*号',
    full_text,
    re.DOTALL,  # 允许跨行匹配
)
```

**效果**: ✅ 成功提取 "皖(2025)蚌埠市不动产权"（部分）

---

#### ✅ 修复5: 购房款(小写)
**问题**: 文档中"小写XXX元"在"购房款"之前，且跨行  
**解决方案**: 添加反向匹配模式 + 使用 `[\s\S]` 支持跨行

```python
# 模式3: 处理顺序反转 "小写XXX元...购房款" (支持跨行)
match = re.search(r'[（(]小写\s*([\d,.]+)\s*元[)）][\s\S]*?购房款', full_text)
```

**效果**: ✅ 成功提取 "450000.00"

## 6. 问题与解决方案

### 6.1 遇到的问题

**问题1: 跨行匹配失败**
- **现象**: 不动产权证号和购房款(小写)无法匹配
- **根因**: OCR文本被换行分割，`[^\n]` 无法跨行匹配
- **解决**: 使用 `re.DOTALL` 和 `[\s\S]` 支持跨行匹配
- **效果**: ✅ 成功修复

**问题2: 文档格式变体多**
- **现象**: 不同案例使用不同的表述方式
- **根因**: 正则表达式过于严格，无法覆盖所有变体
- **解决**: 添加多个备选模式（模式1、模式2、模式3）
- **效果**: ✅ 成功修复

### 6.2 未解决的问题

**问题: OCR质量差**
- **现象**: 贷款字段识别为"null"或缺失
- **根因**: OCR引擎无法识别手写体或印刷质量差的文本
- **解决**: 需要VLM兜底或人工校对
- **状态**: ⏳ 留待第二阶段
