#### 核心类：`OCRTextPreprocessor`

```python
class OCRTextPreprocessor:
    """OCR文本预处理器"""

    def __init__(self, enable_cleaning: bool = True, enable_standardization: bool = True):
        """
        初始化预处理器

        Args:
            enable_cleaning: 是否启用文本清理
            enable_standardization: 是否启用格式标准化
        """
```

#### 功能模块

##### 1.1 文本清理 (`_clean_text`)

**功能**：
- 移除零宽字符和控制字符
- 全角空格转半角空格
- 规范化空白字符（制表符、换页符等）
- 移除行首行尾多余空格
- 压缩连续多个空格为单个空格
- 压缩连续多个换行为单个换行

**示例**：
```python
"2026年 1月 14日" → "2026年1月14日"
"房屋\t坐落\n龙湖嘉园" → "房屋 坐落\n龙湖嘉园"
"房屋坐落　龙湖嘉园" → "房屋坐落 龙湖嘉园"
"房屋  坐落   龙湖嘉园" → "房屋 坐落 龙湖嘉园"
```

##### 1.2 日期标准化 (`_standardize_dates`)

**功能**：
- 移除日期中的多余空格
- 统一日期格式

**示例**：
```python
"2026年 1月 14日" → "2026年1月14日"
"2026 年 1 月 14 日" → "2026年1月14日"
"2026- 01- 14" → "2026-01-14"
"2026 - 01 - 14" → "2026-01-14"
"2026/ 01/ 14" → "2026/01/14"
```

##### 1.3 金额标准化 (`_standardize_amounts`)

**功能**：
- 统一货币符号格式
- 移除货币符号后的空格

**示例**：
```python
"¥ 40000.00" → "¥40000.00"
"￥ 40000.00" → "¥40000.00"
```

##### 1.4 标点符号标准化 (`_standardize_punctuation`)

**功能**：
- 移除标点符号后的多余空格
- 保持中文标点符号

**示例**：
```python
"房屋坐落，龙湖嘉园" → "房屋坐落，龙湖嘉园"
"房屋坐落， 龙湖嘉园" → "房屋坐落，龙湖嘉园"
```

## 🎯 预处理模块价值

### 1. 预防性维护

虽然当前测试案例中预处理没有带来额外提升，但为未来可能出现的新格式提供了容错能力：

**场景1**：OCR识别出带空格的日期
```
"2026年 1月 14日" → 预处理 → "2026年1月14日" → 正则匹配成功
```

**场景2**：OCR识别出带空格的金额
```
"¥ 40000.00" → 预处理 → "¥40000.00" → 正则匹配成功
```

### 2. 代码整洁

预处理模块将文本清理逻辑集中管理，使正则表达式更简洁：

**优化前**：
```python
# 需要在多个正则表达式中处理空格
match = re.search(r'(\d{4})\s*年\s*(\d{1,2})\s*月\s*(\d{1,2})\s*日', full_text)
```

**优化后**：
```python
# 预处理已移除空格，正则更简洁
match = re.search(r'(\d{4})年(\d{1,2})月(\d{1,2})日', full_text)
```

### 3. 可维护性

所有文本清理逻辑集中在 `text_preprocessor.py` 中，便于后续优化和扩展：

- 新增清理规则：只需修改预处理模块
- 调整清理策略：只需修改预处理模块
- 不影响现有正则表达式逻辑

### 4. 扩展性

为后续更复杂的OCR错误纠正提供基础框架：

**未来可扩展功能**：
- 常见OCR错别字纠正（如"己"→"已"）
- 数字格式统一（如"１２３"→"123"）
- 特殊字符规范化（如"①"→"1"）
