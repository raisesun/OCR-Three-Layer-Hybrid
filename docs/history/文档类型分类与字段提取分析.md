## 一、业务背景

本系统服务于**安徽省不动产交易税收综合管理系统**，处理两类核心业务：

| 业务类型 | 说明 | 样本文件夹数 | 预估图片数 |
|---------|------|-------------|-----------|
| **存量房**（二手房） | 已有产权证的房屋交易 | 20个（BBJZ系列） | ~600张 |
| **增量房**（商品房） | 开发商新房销售 | 20个（日期编码系列） | ~400张 |

**总计：969张图片，40个业务案例**

## 二、文档类型详细分析

### 2.1 居民身份证（正面+背面）

**文档特征：**
- 国家标准格式，布局完全固定
- 正面：照片 + 姓名/性别/民族/出生/住址/身份证号
- 背面：签发机关 + 有效期限

**提取字段：**

| 字段名 | 字段说明 | 示例值 |
|--------|---------|--------|
| 姓名 | 持证人姓名 | 张三 |
| 性别 | 男/女 | 男 |
| 民族 | 民族名称 | 汉 |
| 出生 | 出生日期 | 1990年1月1日 |
| 住址 | 户籍地址 | 安徽省合肥市XX区XX路XX号 |
| 公民身份号码 | 18位身份证号 | 340104199001011234 |
| 签发机关 | 发证单位（背面） | 合肥市公安局 |
| 有效期限 | 证件有效期（背面） | 2010.01.01-2030.01.01 |

**样本统计：**
- 正面：~19张
- 背面：~14张
- 分布：存量房和增量房均有

**分类建议：🟢 固定文档（规则层）**
- 理由：全国标准模板，字段位置固定，正则表达式可提取
- 预期：准确率100%，响应<1秒

---

### 2.2 户口本（常住人口登记卡）

**文档特征：**
- 表格结构，包含多个家庭成员
- 各省格式略有差异（安徽/江苏/浙江等）
- 包含户主页 + 多个家庭成员页

**提取字段：**

| 字段名 | 字段说明 | 示例值 |
|--------|---------|--------|
| 户别 | 农业/非农业 | 非农业家庭户 |
| 户号 | 户口本编号 | 340104001234 |
| 住址 | 户籍地址 | 安徽省合肥市XX区XX路XX号 |
| 全户人数 | 家庭成员总数 | 4 |
| 姓名 | 成员姓名 | 张三 |
| 与户主关系 | 家庭关系 | 户主/配偶/子女 |
| 户籍 | 户籍性质 | 安徽省合肥市 |
| 出生日期 | 成员出生日期 | 1990年1月1日 |
| 公民身份编号 | 身份证号 | 340104199001011234 |
| 文化程度 | 学历 | 本科 |
| 婚姻状况 | 婚姻状态 | 已婚 |
| 服务处所 | 工作单位 | XX公司 |

**样本统计：**
- 数量：~32张（占比26.7%，最多）
- 分布：存量房和增量房均有

**分类建议：🟡 半固定文档（VLM层）**
- 理由：
  - 表格结构，成员数量不固定（1-10人）
  - 各省格式有差异
  - 当前基准测试匹配率仅20.5%（规则层失败）
  - 需要VLM理解表格结构
- 挑战：多成员提取、表格结构识别

### 2.3 结婚证

**文档特征：**
- 国家标准格式，但图片质量差异大
- 包含夫妻双方信息
- 带有公章和照片

**提取字段：**

| 字段名 | 字段说明 | 示例值 |
|--------|---------|--------|
| 持证人 | 证件持有人 | 张三 |
| 姓名（男方） | 男方姓名 | 张三 |
| 姓名（女方） | 女方姓名 | 李四 |
| 性别 | 双方性别 | 男/女 |
| 出生日期（男方） | 男方生日 | 1990年1月1日 |
| 出生日期（女方） | 女方生日 | 1992年5月5日 |
| 身份证号（男方） | 男方身份证号 | 340104199001011234 |
| 身份证号（女方） | 女方身份证号 | 340104199205055678 |
| 结婚登记日期 | 登记日期 | 2015年10月1日 |
| 结婚证字号 | 证件编号 | J340104-2015 |
| 登记机关 | 发证单位 | 合肥市XX区民政局 |

**样本统计：**
- 数量：~12张（占比10%）
- 分布：主要在存量房（用于证明婚姻状况）

**分类建议：🟢 固定文档（规则层）**
- 理由：
  - 全国标准模板，字段位置固定
  - 可用正则表达式提取
- 注意事项：
  - 当前基准测试匹配率仅25%（图片质量问题）
  - 建议：先尝试规则层，如效果不佳可升级到VLM层

### 2.4 房产证 / 不动产权证

**文档特征：**
- 政府颁发的产权证书
- 包含房屋基本信息和权利人信息
- 不同版本的证书格式有差异（旧版房产证 vs 新版不动产权证）

**提取字段：**

| 字段名 | 字段说明 | 示例值 |
|--------|---------|--------|
| 权利人 | 房屋所有权人 | 张三 |
| 共有情况 | 单独所有/共同共有 | 单独所有 |
| 坐落 | 房屋地址 | 合肥市XX区XX路XX号XX室 |
| 不动产单元号 | 唯一标识号 | 340104001001GB00001F00010001 |
| 房屋性质 | 商品房/房改房等 | 商品房 |
| 规划用途 | 住宅/商业等 | 住宅 |
| 面积 | 建筑面积（平方米） | 89.5 |
| 权利类型 | 国有建设用地使用权/房屋所有权 | 房屋所有权 |
| 权利性质 | 出让/划拨 | 出让 |

**样本统计：**
- 数量：~18张（占比15%）
- 分布：主要在存量房

**分类建议：🟡 半固定文档（VLM层）**
- 理由：
  - 当前基准测试匹配率仅26.9%
  - 新旧版本格式不同
  - 字段位置相对固定，但内容长度变化大
  - VLM可更好理解表格和长文本

### 2.5 购房合同 / 商品房买卖合同

**文档特征：**
- 多页法律文件（通常10-30页）
- 密集的法律条款和表格
- 包含买卖双方信息、房屋信息、价格条款等

**提取字段：**

| 字段名 | 字段说明 | 示例值 |
|--------|---------|--------|
| 合同编号 | 合同唯一编号 | 202401001 |
| 出卖人 | 卖方（开发商） | 合肥XX房地产开发有限公司 |
| 买受人 | 买方（购房者） | 张三 |
| 项目名称 | 楼盘名称 | XX花园 |
| 房屋地址 | 具体房号 | XX花园X幢X室 |
| 建筑面积 | 面积（平方米） | 89.5 |
| 单价 | 每平方米价格 | 15000元/㎡ |
| 总金额 | 合同总价 | 1342500元 |
| 付款方式 | 一次性/按揭 | 按揭贷款 |
| 交付日期 | 交房时间 | 2025年12月31日 |

**样本统计：**
- 数量：~19张（占比15.8%）
- 分布：主要在增量房

**分类建议：🔴 复杂文档（LLM层）**
- 理由：
  - 长文档（可能>10页，>10K token）
  - 法律条款复杂
  - 当前基准测试匹配率仅35.3%
  - 需要LLM理解上下文才能准确提取
- 挑战：响应时间可能>20秒

## 三、文档分类汇总

### 3.1 三层架构分类

```
┌─────────────────────────────────────────────────────────┐
│                    输入图片                               │
│                  （969张）                               │
└──────────────────┬──────────────────────────────────────┘
                   │
                   ▼
          ┌────────────────┐
          │  文档分类器     │
          │ （关键词匹配）  │
          └────────┬───────┘
                   │
    ┌──────────────┼──────────────┐
    │              │              │
    ▼              ▼              ▼
┌────────┐    ┌────────┐    ┌────────┐
│规则层   │    │VLM层   │    │LLM层   │
│(固定)   │    │(半固定) │    │(复杂)   │
└────────┘    └────────┘    └────────┘
```

### 3.2 分类结果

| 层级 | 文档类型 | 数量 | 占比 | 目标准确率 | 目标耗时 |
|------|---------|------|------|-----------|---------|
| **🟢 规则层** | 身份证（正/背） | ~33张 | 27.5% | 100% | <1秒 |
| | 结婚证 | ~12张 | 10.0% | 100% | <1秒 |
| | 增值税发票 | 多张 | ~5% | 100% | <1秒 |
| | **小计** | ~50张 | **42.5%** | | |
| **🟡 VLM层** | 户口本 | ~32张 | 26.7% | 99% | <15秒 |
| | 房产证/不动产权证 | ~18张 | 15.0% | 99% | <15秒 |
| | 房屋交易协议 | 少量 | ~2% | 99% | <15秒 |
| | **小计** | ~52张 | **43.7%** | | |
| **🔴 LLM层** | 购房合同 | ~19张 | 15.8% | 99% | <20秒 |
| | 预售合同 | 少量 | ~2% | 99% | <20秒 |
| | **小计** | ~21张 | **17.8%** | | |

**加权平均目标：**
- 准确率：42.5%×100% + 43.7%×99% + 17.8%×99% = **99.1%**
- 耗时：42.5%×1秒 + 43.7%×15秒 + 17.8%×20秒 = **10.6秒**

## 五、待确认事项

### 5.1 需要讨论的问题

1. **分类器设计**
   - 关键词匹配是否足够？
   - 是否需要加入图像特征（尺寸、颜色）？
   - 如何处理边界案例（如身份证+户口本同框）？

2. **VLM层挑战**
   - 户口本匹配率仅20.5%，如何提升？
   - 是否需要收集更多省份的样本？
   - 是否需要设计专门的prompt模板？

3. **LLM层挑战**
   - 购房合同可能超过10页，如何处理？
   - 是否需要分页提取后合并？
   - 响应时间可能超过20秒，是否接受？

4. **质量兜底**
   - 低置信度结果如何处理？
   - 是否需要人工审核环节？
   - 如何设计置信度阈值？

5. **样本代表性**
   - 969张样本是否覆盖所有场景？
   - 是否需要补充边缘案例（老旧证件、损坏证件）？
   - 各省证件格式差异是否覆盖充分？
