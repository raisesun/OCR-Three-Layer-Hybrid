# 50样本全量评测报告

**评测日期**：2026-07-01  
**评测样本**：50个（覆盖9种文档类型）  
**评测模型**：GLM-OCR vs Qwen2.5-VL-7B  
**总耗时**：约1.5小时

## 执行摘要

本次评测对比了GLM-OCR和Qwen2.5-VL-7B两个VLM模型在50个真实样本上的表现。结果显示两个模型的提取效果完全相同（成功率82%，平均字段2.2，准确率75%），但Qwen2.5-VL-7B在速度上有明显优势（快35%）。

**核心结论**：推荐Qwen2.5-VL-7B作为生产环境VLM引擎。

## 总体结果

| 指标 | GLM-OCR | Qwen2.5-VL-7B | 差异 |
|------|---------|---------------|------|
| 成功率 | 82.0% | 82.0% | 相同 |
| 平均字段数 | 2.2 | 2.2 | 相同 |
| 字段准确率 | 75.0% | 75.0% | 相同 |
| 平均耗时 | 61.4秒 | 39.9秒 | **Qwen快35%** ✅ |
| 总耗时 | 51.1分钟 | 33.3分钟 | **Qwen快35%** ✅ |

| 处理层 | GLM-OCR | Qwen2.5-VL-7B | 加速比 |
|--------|---------|---------------|--------|
| rule层 | 68.4秒 | 23.2秒 | **2.9倍** ✅ |

**分析**：
- rule层速度提升最明显（2.9倍），因为Qwen2.5-VL-7B的OCR预处理更快

- llm层速度持平（两个模型的LLM调用相同）

## 问题诊断

### 问题1：LLM层处理器未配置

**现象**：9个样本因"没有可用的llm层处理器"失败

**原因**：
- `service.py`中LLM层处理器未正确初始化
- 或者配置中禁用了LLM层

**影响**：
- 规则层和VLM层提取失败时，无法回退到LLM层
- 导致7个购房合同样本失败

**解决方案**：
1. 检查`config.py`中`enable_llm_extraction`配置
2. 检查`service.py`中LLM层处理器初始化逻辑
3. 确保PP-ChatOCRv4服务可用（端口11434）

### 问题2：购房合同提取失败

**现象**：购房合同成功率仅22%（2/9）

**原因**：
1. 规则层未能识别购房合同的关键字段（买受人、出卖人、总价款等）
2. VLM层也未能提取到有效字段
3. LLM层未配置，无法回退

**解决方案**：
1. 优化规则层的购房合同提取逻辑
2. 增加购房合同的关键词匹配
3. 配置LLM层处理器作为兜底

### 问题3：VLM层提取字段数为0

**现象**：10次VLM层调用全部成功，但字段数为0

**原因**：
- VLM层返回了结果，但未能提取到指定字段
- 可能是因为字段名不匹配，或者VLM返回格式不符合预期

**解决方案**：
1. 检查VLM层的字段提取逻辑
2. 优化VLM的prompt，提高字段提取准确率
3. 增加字段名映射和模糊匹配
