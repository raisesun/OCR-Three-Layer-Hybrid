# OCR系统扩展指南 — 新增证件类型

## 概述

本文档详细说明如何在现有OCR三层混合架构中新增证件类型和单证类型。

### 扩展示例

- **证件类型**: 驾驶证、行驶证、银行卡
- **单证类型**: 空运提单、快递单

### 4.3 实现具体提取方法

```python
def _extract_driver_license(self, full_text: str, key_list: List[str]) -> Dict[str, str]:
    """
    提取驾驶证字段
    
    字段示例：
    - 姓名
    - 性别
    - 国籍
    - 住址
    - 出生日期
    - 初次领证日期
    - 准驾车型
    - 有效期限
    - 证号（身份证号）
    """
    fields = {}
    
    # 姓名：通常有"姓名"标签
    name_match = re.search(r'姓名\s*([^\s]+)', full_text)
    if name_match:
        fields['姓名'] = name_match.group(1)
    
    # 准驾车型：通常有"准驾车型"标签
    class_match = re.search(r'准驾车型\s*([A-Za-z0-9]+)', full_text)
    if class_match:
        fields['准驾车型'] = class_match.group(1)
    
    # 证号（身份证号）：18位数字
    id_match = re.search(r'证号\s*(\d{17}[\dXx])', full_text)
    if id_match:
        fields['证号'] = id_match.group(1)
    
    # ... 其他字段 ...
    
    return fields


def _extract_vehicle_license(self, full_text: str, key_list: List[str]) -> Dict[str, str]:
    """
    提取行驶证字段
    
    字段示例：
    - 号牌号码
    - 车辆类型
    - 所有人
    - 住址
    - 使用性质
    - 品牌型号
    - 车辆识别代号（VIN）
    - 发动机号码
    - 注册日期
    - 发证日期
    """
    fields = {}
    
    # 号牌号码
    plate_match = re.search(r'号牌号码\s*([^\s]+)', full_text)
    if plate_match:
        fields['号牌号码'] = plate_match.group(1)
    
    # 车辆识别代号（VIN）：17位字母数字
    vin_match = re.search(r'车辆识别代号\s*([A-HJ-NPR-Z0-9]{17})', full_text)
    if vin_match:
        fields['车辆识别代号'] = vin_match.group(1)
    
    # ... 其他字段 ...
    
    return fields


def _extract_bank_card(self, full_text: str, key_list: List[str]) -> Dict[str, str]:
    """
    提取银行卡字段
    
    字段示例：
    - 卡号
    - 持卡人姓名
    - 有效期
    - 发卡银行
    """
    fields = {}
    
    # 卡号：通常16-19位数字
    card_match = re.search(r'(\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4,7})', full_text)
    if card_match:
        fields['卡号'] = card_match.group(1).replace(' ', '').replace('-', '')
    
    # ... 其他字段 ...
    
    return fields


def _extract_air_waybill(self, full_text: str, key_list: List[str]) -> Dict[str, str]:
    """
    提取空运提单字段
    
    字段示例：
    - 运单号
    - 始发站
    - 目的站
    - 托运人
    - 收货人
    - 航班号
    - 日期
    - 件数
    - 重量
    """
    fields = {}
    
    # 运单号：通常11位数字（3位航司代码 + 8位序列号）
    awb_match = re.search(r'(?:运单号|AWB|Air Waybill)\s*(\d{3}-?\d{8})', full_text)
    if awb_match:
        fields['运单号'] = awb_match.group(1)
    
    # ... 其他字段 ...
    
    return fields


def _extract_express_waybill(self, full_text: str, key_list: List[str]) -> Dict[str, str]:
    """
    提取快递单字段
    
    字段示例：
    - 快递单号
    - 寄件人姓名
    - 寄件人电话
    - 寄件地址
    - 收件人姓名
    - 收件人电话
    - 收件地址
    - 快递公司
    """
    fields = {}
    
    # 快递单号：根据快递公司不同，长度不同
    tracking_match = re.search(r'(?:快递单号|单号|Tracking)\s*([A-Za-z0-9]{10,20})', full_text)
    if tracking_match:
        fields['快递单号'] = tracking_match.group(1)
    
    # 寄件人姓名
    sender_match = re.search(r'寄件人[:\s]*([^\s,，]+)', full_text)
    if sender_match:
        fields['寄件人姓名'] = sender_match.group(1)
    
    # 收件人姓名
    receiver_match = re.search(r'收件人[:\s]*([^\s,，]+)', full_text)
    if receiver_match:
        fields['收件人姓名'] = receiver_match.group(1)
    
    # ... 其他字段 ...
    
    return fields
```

## 常见问题

### Q1: 应该选择规则层还是VLM层？

| 场景 | 推荐层 | 原因 |
|------|--------|------|
| 格式固定、字段位置固定 | 规则层 | 速度快、准确率高 |
| 格式多样、布局复杂 | VLM层 | 灵活性强 |
| 需要快速上线 | VLM层 | 只需写Prompt |
| 追求极致性能 | 规则层 | 毫秒级响应 |

### Q2: 如何处理多页文档？

对于多页文档（如驾驶证正反面），在 `_detect_page_type()` 中添加页面识别逻辑：

```python
def _detect_page_type(self, doc_type, full_text):
    if doc_type == DocumentType.DRIVER_LICENSE:
        if "准驾车型" in full_text:
            return PageType.CONTENT  # 正面
        elif "检验记录" in full_text:
            return PageType.BACK     # 背面
    return PageType.UNKNOWN
```

### Q3: 如何处理不同版本的证件？

使用备选信号（Stage 1.5）来处理不同版本：

```python
BACKUP_CERTIFICATE_SIGNALS = {
    DocumentType.DRIVER_LICENSE: {
        "primary": ["机动车驾驶证"],  # 新版本
        "required": ["准驾车型"],
    },
}
```

## 总结

扩展现有OCR系统支持新证件类型只需7个步骤：

1. **定义枚举** (interfaces.py)
2. **添加分类规则** (classifier.py)
3. **配置路由** (pipeline.py)
4. **实现规则层提取** (rule_layer.py)
5. **实现VLM层Prompt** (vlm_layer.py)
6. **添加字段列表** (pipeline.py)
7. **测试验证**

每个步骤都是独立的，可以根据需要选择性实现。如果时间紧迫，可以先实现VLM层（只需写Prompt），后续再补充规则层以提升性能。
