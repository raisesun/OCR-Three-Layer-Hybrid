## 1. 项目背景

### 1.1 业务场景

存量房交易资金监管业务需要处理多种文档类型：
- **资金监管协议**（多页文档）
  - 首页：基本信息、房屋信息、金额信息
  - 信息页：甲乙方详细信息
  - 签章页：签字盖章
- **资金监管凭证**：交易完成凭证

### 1.2 技术架构

采用OCR三层混合架构 v2.0：
```
图片输入 → OCR识别 → 文档分类 → 字段提取
                    ↓
              规则层（正则表达式）
              VLM层（大模型兜底）
```

### 1.3 优化目标

- 提高资金监管协议首页字段提取完成率
- 正确处理"无贷款"业务场景
- 优化字段定义，符合业务需求

## 2. 问题诊断

### 2.1 初始测试结果

**测试案例**: BBJZ-2026-0112065 首页

**提取结果**:
```
预期字段: 13个
实际提取: 6个
完成率: 46%
```

**缺失字段**:
- ❌ 签署日期
- ❌ 网上签约备案合同号
- ❌ 房屋地址
- ❌ 不动产权证号
- ❌ 购房款(小写)
- ❌ 贷款(大写)
- ❌ 贷款(小写)

### 2.2 根因分析

#### 问题1: OCR文本格式差异

**实际OCR文本**:
```
第一条 甲、乙双方于 2026年1月12日 签订《存量房买卖合同》
...
网上签约备案合同号为：BBJZ-2026-0112065
...
甲方自愿将位于蚌埠市安瑞科嘉苑1号楼1单元11层4号的房屋出售给乙方
...
原产权证或不动产权证证号为：皖(2025)蚌埠市不动产权第0081745号
...
乙方将购房款人民币肆拾伍万元整（小写 450000.00 元）存入丙方账户
...
贷款人民币      元（小写 null 元）
```

**发现的问题**:
1. 日期格式：`2026年1月12日`（标准格式）
2. 合同号前缀：`网上签约备案合同号为：`（带"为"字）
3. 地址前缀：`位于`（非标准"房屋地址："）
4. 证号前缀：`证号为：`（非标准"不动产权证号："）
5. 金额格式：`小写 450000.00 元`（数字在小写后）
6. 贷款字段：`小写 null 元`（表示无贷款）

#### 问题2: 正则表达式过于严格

**原始正则表达式**:
```python
# 签署日期
r'签署日期\s*[：:]\s*(\d{4}年\d{1,2}月\d{1,2}日)'

# 网上签约备案合同号
r'(?:网上签约备案合同号|合同号)[：:]\s*([A-Z0-9\-]+)'

# 房屋地址
r'房屋地址[：:]\s*([^\n]+)'

# 不动产权证号
r'不动产权证号[：:]\s*([^\n]+)'

# 购房款(小写)
r'购房款\s*[\(（]小写[\)）]\s*[：:]*\s*[¥￥]?\s*([\d,]+\.?\d*)'
```

**问题分析**:
1. 签署日期：要求"签署日期"前缀，但实际是"于...签订"
2. 合同号：要求直接跟"合同号："，但实际是"合同号为："
3. 房屋地址：要求"房屋地址："前缀，但实际是"位于"
4. 证号：要求"不动产权证号："前缀，但实际是"证号为："
5. 小写金额：要求"购房款（小写）"，但实际是分开出现

#### 问题3: 业务逻辑理解错误

**原始字段定义**:
```python
DocumentType.FUND_SUPERVISION_AGREEMENT_FIRST_PAGE: [
    "编号", "甲方", "乙方", "丙方",
    "签署日期", "网上签约备案合同号",
    "房屋地址", "建筑面积", "不动产权证号",
    "购房款(大写)", "购房款(小写)",
    "贷款(大写)", "贷款(小写)",  # 全部视为必填
]
```

**业务问题**:
- 贷款字段被标记为必填
- 实际业务中，"null"或空值表示"无贷款"，是有效数据
- 缺少统一的"购房款"字段

## 3. 优化方案

### 3.1 正则表达式优化策略

**核心思路**: 增加多个备选模式，覆盖不同格式变体

**优化方法**:
1. **添加备选模式**: 为每个字段提供2-3个正则表达式
2. **支持跨行匹配**: 使用 `[\s\S]*?` 替代 `[^\n]*?`
3. **可选前缀**: 使用 `(?:为)?` 使前缀可选
4. **模糊匹配**: 允许更多格式变体

### 3.2 字段定义调整策略

**核心思路**: 区分必填字段和可选字段，符合业务逻辑

**调整方法**:
1. **新增"购房款"字段**: 统一表示购房金额
2. **标记贷款字段为可选**: 空值/null表示无贷款
3. **保持向后兼容**: 保留原有字段定义

**提取逻辑**: `src/ocr_three_layer_hybrid/rule_layer.py`

```python
def _extract_fund_supervision_agreement_first_page(self, full_text, key_list):
    fields = {}
    
    # 提取购房款（优先小写）
    if "购房款" in key_list:
        # 优先提取小写金额
        match = re.search(r'[\(（]小写[\)）]\s*[：:]*\s*[¥￥]?\s*([\d,]+\.?\d*)', full_text)
        if match:
            fields["购房款"] = match.group(1)
        else:
            # 尝试提取大写金额（转换为数字）
            match = re.search(r'购房款[：:]\s*([^（\n]+)', full_text)
            if match:
                fields["购房款"] = self._chinese_to_number(match.group(1).strip())
    
    return fields
```

#### 4.2.2 贷款字段标记为可选

**修改文件**: `src/ocr_three_layer_hybrid/rule_layer.py`

```python
def _extract_fund_supervision_agreement_first_page(self, full_text, key_list):
    fields = {}
    
    # 提取贷款字段（可选）
    for field in ["贷款(大写)", "贷款(小写)"]:
        if field in key_list:
            # 检查是否为null或空值
            if "null" in full_text or re.search(r'贷款[^\n]*[\(（]小写[\)）]\s*[：:]*\s*[¥￥]?\s*$', full_text):
                fields[field] = ""  # 空值表示无贷款
            else:
                # 正常提取
                match = re.search(r'贷款[^\n]*[\(（]小写[\)）]\s*[：:]*\s*[¥￥]?\s*([\d,]+\.?\d*)', full_text)
                if match:
                    fields[field] = match.group(1)
                else:
                    fields[field] = ""  # 未找到，默认为空
    
    return fields
```

## 8. 经验总结

### 8.1 成功经验

#### 8.1.1 问题诊断方法

**关键做法**:
1. **深入分析OCR文本**: 不假设格式，实际查看OCR输出
2. **逐个字段分析**: 针对每个缺失字段找出根因
3. **多案例验证**: 在不同案例上验证问题普遍性

**效果**:
- 准确定位问题根因
- 避免错误假设
- 提高优化针对性

#### 8.1.2 正则表达式优化策略

**关键做法**:
1. **多模式匹配**: 为每个字段提供多个备选模式
2. **渐进式优化**: 从最严格到最宽松
3. **跨行匹配**: 支持OCR文本换行
4. **充分测试**: 在每个案例上验证效果

**效果**:
- 覆盖更多格式变体
- 提高提取成功率
- 保持代码可维护性

#### 8.1.3 业务逻辑理解

**关键做法**:
1. **与业务方沟通**: 理解"null"值的业务含义
2. **区分必填/可选**: 根据业务需求定义字段
3. **验证业务场景**: 确保优化符合实际业务

**效果**:
- 字段定义更符合业务需求
- 正确处理特殊场景
- 提高数据质量

### 8.2 教训总结

#### 8.2.1 不要假设OCR格式

**问题**:
- 初始假设OCR输出格式统一
- 实际发现多种格式变体

**教训**:
- 必须先查看实际OCR输出
- 不能基于假设编写正则表达式
- 要预留足够的格式灵活性

#### 8.2.2 不要忽视业务逻辑
