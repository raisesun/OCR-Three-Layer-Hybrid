# 预处理调优最终报告

**日期**: 2026-07-02  
**测试图片**: 身份证 (刘开顺)  
**结论**: 仅去噪预处理 + 规则层优化 = 100%准确率

---

## 1. 问题回顾

### 初始问题
- 预处理（去噪+对比度增强）导致姓名和住址提取失败
- 准确率从83.3%下降到66.7%

### 根因分析
1. **OCR输出格式变化**：预处理后OCR更倾向于输出"标签+值"格式
2. **规则层正则不匹配**：原有正则针对"值+标签"格式优化，无法匹配新格式
3. **具体错误**：
   - 姓名正则匹配到"性别男\n姓名"，提取出"性别男"
   - 住址正则无法处理多行地址（被其他字段打断）

---

## 2. 解决方案

### 2.1 规则层正则优化

**修复1：姓名提取**
```python
# 修改前：格式1优先（值+标签）
match = re.search(r'([一-鿿]{2,4})\s*\n\s*姓名', full_text)
# 匹配到"性别男\n姓名" → 错误

# 修改后：格式1优先（标签+同行值）
match = re.search(r'(?<!户主)姓\s*名\s*[:：]?\s*([一-鿿]{2,4})(?=\s|$)', full_text)
# 匹配到"姓名刘开顺" → 正确
```

**修复2：住址提取**
```python
# 修改前：期望连续的多行地址
match = re.search(r'住址\s*([一-鿿]+(?:省|市|区|县|镇|乡|村|路|号|室)[^\n]*(?:\n...)*)', full_text)
# 无法处理中间有其他字段的地址

# 修改后：允许地址中间有其他字段
match = re.search(r'住址\s*([一-鿿]+(?:省|市|区|县|镇|乡|村|路|号|室)[^\n]*)', full_text)
# 查找后续行中的地址剩余部分
if match:
    candidate = match.group(1).strip()
    lines_after = full_text[match.end():].split('\n')
    for line in lines_after[:5]:
        if (any(kw in line for kw in ['镇', '乡', '村', '路', '号', '室']) and
            not any(label in line for label in ['出生', '性别', '民族', ...])):
            candidate += line
            break
```

### 2.2 预处理配置优化

**推荐配置**：仅去噪
```python
config = OCRConfig(
    enable_image_preprocessing=True,
    preprocessing_denoise=True,       # 启用去噪
    preprocessing_deskew=False,       # 禁用纠偏（对竖向文档误判）
    preprocessing_contrast=False,     # 禁用对比度增强（导致格式混乱）
    preprocessing_binarize=False,
)
```

**为什么禁用对比度增强**：
- 对比度增强导致OCR输出格式混乱
- 标签和值的相对位置被打乱
- 地址被拆分成多行且顺序颠倒

## 4. 关键发现

### 4.1 预处理的影响因素

| 预处理组件 | 对OCR的影响 | 对提取的影响 |
|------------|-------------|--------------|
| 去噪 | 提高标签识别率 | ✓ 正面（格式更清晰） |
| 对比度增强 | 改变标签-值相对位置 | ✗ 负面（格式混乱） |
| 纠偏 | 对竖向文档误判90° | ✗ 负面（数据丢失） |
| 二值化 | 丢失灰度信息 | ✗ 负面（细节丢失） |

### 4.2 仅去噪的优势

1. **格式更清晰**：标签+同行值（"姓名刘开顺"）
2. **出生日期完整**：从"1994年3月"提升到"1994年3月1日"
3. **分类置信度提高**：从0.90提升到0.95
4. **准确率100%**：所有字段正确提取

### 核心洞察

1. **预处理不是越多越好**：对比度增强反而降低了准确率
2. **OCR格式决定提取策略**：规则层正则必须匹配OCR输出格式
3. **简单方案可能更有效**：仅去噪比去噪+对比度更好

### 最终建议

**启用仅去噪预处理**，配合已优化的规则层正则，可以达到：
- 100%提取准确率（测试样本）
- 更高的分类置信度
- 更完整的字段提取（如出生日期）

**风险**：仅基于1个样本测试，需要更多样本验证。
