## 四、集成架构验证

### 4.1 三层防御架构

```
┌─────────────────────────────────────────────────────┐
│  第1层: 位置标注提取 (position_extractor.py)         │
│  - 使用PaddleOCR坐标信息                            │
│  - 标签右侧同行搜索策略                              │
│  - 修复列错位、标签合并问题                          │
└─────────────────────────────────────────────────────┘
                        ↓ 失败
┌─────────────────────────────────────────────────────┐
│  第2层: 正则提取 (rule_layer.py)                     │
│  - 传统正则表达式提取                                │
│  - 补充位置标注未覆盖的字段                          │
└─────────────────────────────────────────────────────┘
                        ↓ 校验失败
┌─────────────────────────────────────────────────────┐
│  第3层: VLM兜底 (vlm_fallback.py)                    │
│  - 字段校验器 (field_validator.py)                   │
│  - GLM-OCR视觉模型重新提取                           │
│  - 只覆盖校验失败的字段                              │
└─────────────────────────────────────────────────────┘
```

### 4.2 模块依赖关系

```
service.py
    ├── position_extractor.py (位置标注提取器)
    │       └── PaddleOCR (延迟加载)
    ├── rule_layer.py (规则层)
    │       └── 注入 position_extractor
    ├── field_validator.py (字段校验器)
    ├── vlm_fallback.py (VLM兜底处理器)
    │       └── 使用 field_validator
    └── pipeline.py (编排管道)
            └── 注入 vlm_fallback_handler
```

### 4.3 配置开关

| 配置项 | 默认值 | 说明 |
|-------|--------|------|
| `enable_position_extraction` | True | 启用位置标注提取（户口本首页） |
| `enable_vlm_field_fallback` | True | 启用字段级VLM兜底 |
| `enable_vlm_fallback` | True | 启用VLM分类兜底 |

**当前测试配置**:
- `enable_vlm_field_fallback = False` （VLM模型未下载）
- `enable_position_extraction = True` （已启用）

## 五、已知问题与待办事项

### 5.1 已知问题

#### 问题1: 户别字段提取为空
**现象**: 在集成测试中，户别字段返回空字符串  
**根因**: PaddleOCR将"户别"标签拆分为"户"和"别非农业家庭户"两个独立项  
**解决方案**: 
- 已实现 `_merge_adjacent_items()` 方法合并相邻文本项
- 需要进一步调试合并逻辑是否触发

**状态**: 🔧 调试中

#### 问题2: "王龙晨露"列合并错误
**现象**: OCR将"王晨露"识别为"王龙晨露"（列合并）  
**根因**: PaddleOCR无法正确处理表格列对齐  
**解决方案**: 
- 位置标注无法修复字符级错误
- 需要VLM兜底（代码已完成，等待VLM模型下载）

**状态**: ⏳ 等待VLM模型
