## 二、架构设计对比

### 2.1 ai-ocr-awb 架构

```
PDF文件
  ↓
PyMuPDF渲染（DPI=100，裁剪底部65%）
  ↓
PaddleOCR.predict() → 文本+坐标+置信度
  ↓
坐标规则引擎（V2.9）
  ├─ 锚点定位（关键词：GROSS, KG, VOL等）
  ├─ 多模式正则（每个字段2-3个正则）
  ├─ 一致性校验（cross-validation）
  └─ 字符混淆修复（L→1, S→5等）
  ↓
结构化结果（5个字段）
```

**核心特点**:
- ✅ **纯规则引擎**: 不依赖LLM/VLM，完全基于坐标+正则
- ✅ **锚点定位**: 先找关键词锚点，再推算数值位置
- ✅ **高度优化**: 针对航空运单版式深度优化
- ✅ **速度快**: 2-4秒/文件
- ❌ **扩展性差**: 只能处理航空运单，无法处理其他文档
- ❌ **维护复杂**: 1324行extractor.py，大量硬编码规则

### 2.2 ocr-three-layer-hybrid 架构

```
图片/PDF
  ↓
OCR文本提取（PaddleOCR-VL-0.9B）
  ↓
文档分类器（四阶段路由）
  ├─ 阶段0: 多文档冲突检测
  ├─ 阶段1: 标准证件强信号
  ├─ 阶段2: 标准单证强信号
  ├─ 阶段3: 合同字段组合
  └─ 阶段4: VLM兜底
  ↓
路由决策
  ├─ 固定格式文档 → 规则层
  └─ 半固定/未知文档 → VLM层
  ↓
字段提取
  ├─ 规则层: 正则+位置标注提取
  └─ VLM层: Qwen2.5-VL-7B
  ↓
字段校验（FieldValidator）
  ↓
VLM字段级兜底（可选）
  ↓
结构化结果
```

**核心特点**:
- ✅ **通用架构**: 支持30+种文档类型
- ✅ **三层混合**: 规则层+VLM层+兜底机制
- ✅ **扩展性好**: 新增文档类型只需添加规则
- ✅ **维护简单**: 模块化设计，职责清晰
- ❌ **速度慢**: VLM调用耗时34-89秒
- ❌ **准确率波动**: VLM识别不稳定（60-70%）

## 三、技术实现对比

### 3.1 OCR处理

| 维度 | ai-ocr-awb | ocr-three-layer-hybrid |
|------|-----------|----------------------|
| **OCR引擎** | PaddleOCR 3.x（原生predict） | PaddleOCR-VL-0.9B |
| **输入格式** | PDF（PyMuPDF渲染） | 图片（直接读取） |
| **输出内容** | 文本+坐标+置信度 | 文本+坐标+置信度 |
| **预处理** | 裁剪底部65%（去除页眉） | 无特殊预处理 |
| **坐标系统** | 相对坐标（0-1） | 相对坐标（归一化） |
| **特殊处理** | DPI回退、字符混淆修复 | 位置标注提取 |

**分析**:
- ai-ocr-awb针对PDF优化，使用PyMuPDF渲染
- ocr-three-layer-hybrid针对图片优化，直接处理
- 两者OCR能力相近，都使用PaddleOCR

### 3.2 字段提取方法

#### ai-ocr-awb: 纯规则引擎

```python
# 航空运单字段提取流程
1. 文件名正则 → 提取AWB号（100%准确）
2. 锚点定位 → 找"GROSS"/"KG"关键词
3. 坐标规则 → 在锚点周围搜索数值
4. 多模式正则 → 每个字段2-3个正则，按优先级匹配
5. 一致性校验 → 数字交叉验证
6. 字符修复 → L→1, S→5, I→1等
```

**优势**:
- 准确率极高（99.3%）
- 速度快（2-4秒）
- 不依赖外部服务

**劣势**:
- 只能处理航空运单
- 规则复杂（1324行代码）
- 新文档类型需要大量开发

#### ocr-three-layer-hybrid: 规则+VLM混合

```python
# 快递单字段提取流程（规划）
1. 规则层提取
   ├─ 运单号正则（[A-Za-z0-9]{10,15}）
   ├─ 电话正则（1[3-9]\d{9}）
   ├─ 地址关键词（省市区县镇）
   └─ 姓名提取（2-5个中文字符）

2. VLM层兜底
   ├─ 规则层失败字段 → VLM重新提取
   ├─ 置信度<0.6 → VLM重新提取
   └─ 缺失比例>30% → VLM重新提取

3. 字段校验
   ├─ 格式校验（正则）
   ├─ 长度校验
   └─ 逻辑校验
```

**优势**:
- 通用性强
- 扩展容易
- 模块化设计

**劣势**:
- VLM速度慢（34-89秒）
- 准确率波动（60-70%）
- 依赖外部VLM服务

### 3.3 兜底机制

| 维度 | ai-ocr-awb | ocr-three-layer-hybrid |
|------|-----------|----------------------|
| **兜底层级** | 无（纯规则） | VLM字段级兜底 |
| **触发条件** | N/A | 校验失败/置信度低/缺失比例高 |
| **兜底方式** | N/A | 调用VLM重新提取 |
| **兜底效果** | N/A | 准确率提升10-40% |
| **兜底成本** | 0 | VLM调用成本 |

**分析**:
- ai-ocr-awb不依赖兜底，规则足够精准
- ocr-three-layer-hybrid依赖VLM兜底提升准确率
- 兜底机制增加了复杂度和成本
