## 1. 优化内容

### 1.1 问题发现
初始测试发现签章页识别不完整：
- 预期：每个案例都有签章页
- 实际：只有案例2识别到1个签章页（共1个）
- 缺失：案例1和案例3的签章页未识别

### 1.2 根因分析
通过诊断脚本发现：
1. **签章页结构差异**：
   - 案例2签章页：包含协议内容 + 签章签名（有"资金监管"、"协议"等关键词）
   - 案例1签章页：仅包含签章签名（只有"丙方（签章）"、"签约日期"、"第4页"）

2. **分类器缺陷**：
   - 原逻辑：需要"资金监管" + "监管协议"等关键词
   - 问题：纯签章页缺少这些关键词，无法识别为资金监管文档

### 1.3 修复方案
在 `classifier.py` 中添加**条件5：签章页特殊处理**

```python
# 签章页特殊处理：丙方 + 签章 + (签约日期 或 第X页)
has_丙方 = "丙方" in full_text
has_签章 = "签章" in full_text
has_签约日期 = "签约日期" in full_text
has_第x页 = any(f"第{i}页" in full_text for i in range(1, 10))
is_stamp_page = has_丙方 and has_签章 and (has_签约日期 or has_第x页)
```

**信号组合**：
- "丙方" + "签章" + "签约日期" → 资金监管协议签章页
- "丙方" + "签章" + "第4页" → 资金监管协议签章页

**建议**：
